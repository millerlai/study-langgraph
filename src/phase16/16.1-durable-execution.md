## 1. 什麼是 Durable Execution

### 定義

Durable Execution 是一種技術，讓流程或工作流在**關鍵節點保存進度**，允許它在暫停後**從停止點精確恢復**，而不需要重新處理之前的步驟。

```
┌─────────────────────────────────────────────────────────┐
│                 Durable Execution                        │
│                                                          │
│  正常流程：                                              │
│  [START] → [Node A] → [Node B] → [Node C] → [END]      │
│      ✓ checkpoint  ✓ checkpoint  ✓ checkpoint            ���
│                                                          │
│  失敗後恢復：                                            │
│  [Node B] ← 從最近的 checkpoint 恢復                    │
│      ↓                                                   │
│  [Node C] → [END]                                        │
│      (Node A 不需要重新執行)                              │
│                                                          │
│  Human-in-the-Loop：                                     │
│  [Node A] → [Node B] → interrupt() → 暫停               │
│                              ↓                            │
│                     (一週後恢復)                          │
│                              ↓                            │
│                   [Node B] → [Node C] → [END]            │
└───────���──────────────────────��──────────────────────────┘
```

### 為什麼需要 Durable Execution

| 場景 | 問題 | Durable Execution 解法 |
|------|------|----------------------|
| **LLM Timeout** | API 呼叫超時中斷 | 從失敗前的 checkpoint 恢復 |
| **Human-in-the-Loop** | 需要人類審核後繼續 | interrupt 暫停 → 任意時間後恢復 |
| **長時間任務** | 處理可能跨越數小時/天 | 保存中間進度，隨時恢復 |
| **系統故障** | 伺服器重啟或崩潰 | 從持久化的 checkpoint 恢復 |

### 啟用方式

> **使用 LangGraph 搭配 checkpointer 時，你就已經啟用了 Durable Execution。**

```python
from langgraph.checkpoint.memory import InMemorySaver
from langgraph.graph import StateGraph

builder = StateGraph(State)
# ... 加入 nodes 和 edges ...

# 指定 checkpointer = 啟用 Durable Execution
checkpointer = InMemorySaver()
graph = builder.compile(checkpointer=checkpointer)
```

## 3. 確定性��一致性重播

### 重播機制

當工作流恢復時，程式碼**不會**從中斷的那一行繼續執行。它會：

1. 找到合適的起始點（最近完成的 checkpoint）
2. **重播**從該起始點到中斷點的所有步驟
3. 已完成的 task 結果會從持久化層讀取，而非重新執行

```
恢復執行流程：

原始執行：
[START] → [Node A] → [Node B: task1, task2, task3] → (失敗)
              ✓             ✓     ✓      ✗

恢復執行：
[Node B] ← 從 Node B 開始重播
├── task1: 從 checkpoint 讀取結果（不重新執行）
├── task2: 從 checkpoint 讀取結果（不重新執行）
└── task3: 重新執行（上次失敗了）
```

### 確定性指南

| 原則 | 說明 | 做法 |
|------|------|------|
| **避免重複工作** | 有副作用的操作不應重複執行 | 放入 `@task` 中 |
| **封裝非確定性** | 隨機數、時間戳等不確定值 | 放入 `@task` 或 node 中 |
| **冪等操作** | 相同操作重複執行應有相同效果 | 使用 idempotency key |

### 常見陷阱

```python
# ======= 錯誤做法 =======
def bad_node(state):
    # 問題 1：API 呼叫在 node 內但不在 task 中
    # 恢復時會重複呼叫
    result1 = requests.get("https://api.example.com/data")  # 副作用！

    # ���題 2：隨機值不在 task 中
    # 恢復時會產生不同的值
    random_id = random.randint(1, 1000)  # 非確定性！

    return {"result": result1.text, "id": random_id}


# ======= 正確做法 =======
from langgraph.func import task

@task
def fetch_data(url: str):
    """API 呼叫放入 task，恢復時從 checkpoint 讀取"""
    return requests.get(url).text

@task
def generate_id():
    """隨機值放入 task，恢復時從 checkpoint 讀取"""
    return random.randint(1, 1000)

def good_node(state):
    result_future = fetch_data("https://api.example.com/data")
    id_future = generate_id()
    return {
        "result": result_future.result(),
        "id": id_future.result(),
    }
```

## 4. Durability Modes���持久化模式）

LangGraph 支援三種持久化模式，讓你在效能和資料一致性之間取得平衡：

```
效能（高）                                        持久性（高）
    ◄─────────────────────────────────────────────►
    "exit"              "async"               "sync"
    │                    │                      │
    │ 僅在 graph 結束   │ 非同步寫入           │ 同步寫入
    │ 時保存            │ checkpoint           │ checkpoint
    │                    │ (下一步執行時)      │ (下一步開始前)
    │                    │                      │
    │ 最佳效能          │ 良好效能+持久性      │ 最高持久性
    │ 無中間 state      │ 小機率遺失          │ 保證不遺失
```

### 使用方式

```python
"""
Durability Modes 使用範例
"""
from typing import TypedDict
from langgraph.checkpoint.memory import InMemorySaver
from langgraph.graph import StateGraph, START, END


class State(TypedDict):
    data: str
    result: str


def process(state: State) -> dict:
    return {"result": f"processed: {state['data']}"}


builder = StateGraph(State)
builder.add_node("process", process)
builder.add_edge(START, "process")
builder.add_edge("process", END)

checkpointer = InMemorySaver()
graph = builder.compile(checkpointer=checkpointer)


# === "exit" mode ===
# 僅在 graph 完成（成功/失敗/interrupt）時保存
# 最佳效能，但無法從中間節點恢復
result = graph.invoke(
    {"data": "hello"},
    config={"configurable": {"thread_id": "1"}},
    durability="exit",
)

# === "async" mode（預設） ===
# 非同步保存 checkpoint，下一步開始執行時背景寫入
# 良好的效能和持久性平衡
result = graph.invoke(
    {"data": "hello"},
    config={"configurable": {"thread_id": "2"}},
    durability="async",
)

# === "sync" mode ===
# 同步保存 checkpoint，確保每一步都寫入完成後才繼續
# 最高持久性，但有效能開銷
result = graph.invoke(
    {"data": "hello"},
    config={"configurable": {"thread_id": "3"}},
    durability="sync",
)
```

### 模式選擇指南

| 模式 | 效能 | 持久性 | 適用場景 |
|------|------|--------|----------|
| `"exit"` | 最佳 | 最低 | 短時間 graph、不需中間恢復 |
| `"async"` | 良好 | 良好 | 大多數場景（預設） |
| `"sync"` | 較慢 | 最高 | 關鍵任務、不能遺失任何步驟 |

## 5. 在節��中使用 Task

### 為什麼需要 Task

當一個 node 包含**多個操作**時，將每個操作包裝成 `@task` 可以確保：

1. 恢復時不重複執行已完成的操作
2. 每個操作的結果獨立保存
3. 支援平行執行

```
沒有 Task：
┌──────────────────────────────────┐
│ Node: process_data               │
│                                   │
│  step1: fetch API    ← 恢復時    │
│  step2: transform    ← 全部重新  │
���  step3: save to DB   ← 執行      │
└──────────────────────────────────┘

使用 Task：
┌──────────────────────────────────┐
│ Node: process_data               │
│                                   │
│  @task step1: fetch API  ✓ 已存 │
│  @task step2: transform  ✓ 已存 │
│  @task step3: save to DB ← 只重 │
│                             新執行│
└���─────────────────────────────────┘
```

### 完整範例

```python
"""
在 Node 中使用 @task 的完整範例
確保恢復時不重複執行已完成的副作用操作
"""
import uuid
from typing import NotRequired
from typing_extensions import TypedDict

from langgraph.checkpoint.memory import InMemorySaver
from langgraph.func import task
from langgraph.graph import StateGraph, START, END


class State(TypedDict):
    urls: list[str]
    results: NotRequired[list[str]]


# === 將每個 API 呼叫包裝為 task ===
@task
def fetch_url(url: str) -> str:
    """
    每個 URL 的請求是獨立的 task
    恢復時，已成功的請求不會重新執行
    """
    import requests
    response = requests.get(url)
    return response.text[:200]  # 取前 200 字


@task
def summarize(text: str) -> str:
    """
    摘要也是獨立的 task
    即使這步失敗，之前的 fetch 不需要重做
    """
    # 實際應用中會呼叫 LLM
    return f"Summary of {len(text)} chars: {text[:50]}..."


def process_urls(state: State) -> dict:
    """
    Node 函數：協調多個 task 的執行
    """
    # 平行發起所有 URL 請求
    fetch_futures = [fetch_url(url) for url in state["urls"]]

    # 等待結果並進行摘要
    results = []
    for future in fetch_futures:
        text = future.result()          # 等待 fetch 完成
        summary = summarize(text)        # 發起摘要 task
        results.append(summary.result()) # 等待摘要完成

    return {"results": results}


# === 建構 Graph ===
builder = StateGraph(State)
builder.add_node("process_urls", process_urls)
builder.add_edge(START, "process_urls")
builder.add_edge("process_urls", END)

checkpointer = InMemorySaver()
graph = builder.compile(checkpointer=checkpointer)

# === 執行 ===
thread_id = str(uuid.uuid4())
config = {"configurable": {"thread_id": thread_id}}

result = graph.invoke(
    {"urls": ["https://www.example.com", "https://httpbin.org/get"]},
    config,
)
print(f"結果: {result['results']}")
```

## 6. 故障恢復

### 從異常中恢復

```python
"""
故障恢復範例
展示如何在 node 失敗後從最近的 checkpoint 恢復
"""
import uuid
from typing_extensions import TypedDict
from langgraph.checkpoint.memory import InMemorySaver
from langgraph.graph import StateGraph, START, END

attempt_count = 0


class State(TypedDict):
    input: str
    step1_result: str
    step2_result: str


def step1(state: State) -> dict:
    """第一步：總是成功"""
    print("  執行 step1")
    return {"step1_result": f"step1 processed: {state['input']}"}


def step2(state: State) -> dict:
    """第二步：第一次會失敗，第二次成功"""
    global attempt_count
    attempt_count += 1
    print(f"  執行 step2 (attempt {attempt_count})")

    if attempt_count == 1:
        raise Exception("模擬 LLM API timeout！")

    return {"step2_result": f"step2 processed: {state['step1_result']}"}


# 建構 graph
builder = StateGraph(State)
builder.add_node("step1", step1)
builder.add_node("step2", step2)
builder.add_edge(START, "step1")
builder.add_edge("step1", "step2")
builder.add_edge("step2", END)

checkpointer = InMemorySaver()
graph = builder.compile(checkpointer=checkpointer)

thread_id = str(uuid.uuid4())
config = {"configurable": {"thread_id": thread_id}}

# === 第一次執行 — 會在 step2 失敗 ===
print("=== 第一次執行 ===")
try:
    result = graph.invoke({"input": "hello"}, config)
except Exception as e:
    print(f"  失敗: {e}")

# === 從失敗點恢復 ===
# 使用相同的 thread_id，傳入 None 作為 input
# LangGraph 會從 step2 開始（step1 的結果已保存）
print("\n=== 恢復執行 ===")
result = graph.invoke(None, config)  # None = 從 checkpoint 恢復
print(f"  結果: {result}")
# step1 不會重新執行！只有 step2 會重新執行
```

### 恢復起始點

```
Graph API (StateGraph):
├── 起始點 = 失敗 node 的開頭
├── 如果有子圖呼叫
│   ├── 父 node 從頭開始
│   └── 子圖從失敗的 node 開始
└── 已完成的 node 不會重新執行

Functional API (@entrypoint):
├── 起始點 = entrypoint 的開頭
└── 已完成的 task 結果從 checkpoint 讀取
```

## 7. 關鍵節點進度保存

### Pending Writes（待完成寫入）

當一個 super-step 中有**多個 node 平行執行**，其中一個失敗時：

```
Super-step N:
├── Node A: ✓ 成功 → pending write 已保存
├── Node B: ✓ 成功 → pending write 已保存
└── Node C: ✗ 失敗

恢復時（Super-step N 重新執行）：
├── Node A: 跳過（使用 pending write）
├── Node B: 跳過（使用 pending write）
└── Node C: 重新執行
```

### 最佳實踐

```python
"""
關鍵節點進度保存的最佳實踐
"""
from langgraph.func import task
from langgraph.types import RetryPolicy
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import InMemorySaver
from typing_extensions import TypedDict


class State(TypedDict):
    data: str
    step1: str
    step2: str
    step3: str


# === 最佳實踐 1：將關鍵操作拆為獨立 node ===
# 更多 node = 更頻繁的 checkpoint = 更少的重複工作
def critical_step_1(state: State) -> dict:
    """獨立 node：自動在完成後 checkpoint"""
    return {"step1": "completed"}

def critical_step_2(state: State) -> dict:
    """獨立 node：如果這步失敗，step1 不需重做"""
    return {"step2": "completed"}

def critical_step_3(state: State) -> dict:
    return {"step3": "completed"}


# === 最佳實踐 2：對易失敗的 node 加入 RetryPolicy ===
builder = StateGraph(State)
builder.add_node("step1", critical_step_1)
builder.add_node(
    "step2",
    critical_step_2,
    retry_policy=RetryPolicy(
        max_attempts=3,
        initial_interval=1.0,  # 首次重試等待 1 秒
        backoff_factor=2.0,    # 指數退避
    ),
)
builder.add_node("step3", critical_step_3)

builder.add_edge(START, "step1")
builder.add_edge("step1", "step2")
builder.add_edge("step2", "step3")
builder.add_edge("step3", END)

graph = builder.compile(checkpointer=InMemorySaver())


# === 最佳實踐 3：使用 sync durability 保護關鍵 graph ===
result = graph.invoke(
    {"data": "critical task"},
    config={"configurable": {"thread_id": "critical-1"}},
    durability="sync",  # 每一步都同步寫入
)
```

## 8. 完整範例

```python
"""
Durable Execution 完整示範
模擬一個多步驟資料處理流程，展示故障恢復和 task 使用
"""
import uuid
import time
from typing import NotRequired
from typing_extensions import TypedDict

from langgraph.checkpoint.memory import InMemorySaver
from langgraph.func import task
from langgraph.graph import StateGraph, START, END
from langgraph.types import RetryPolicy


# === State 定義 ===
class PipelineState(TypedDict):
    source_url: str
    raw_data: NotRequired[str]
    cleaned_data: NotRequired[str]
    analysis: NotRequired[str]
    report: NotRequired[str]


# === Task 定義（封裝副作用） ===
@task
def download_data(url: str) -> str:
    """模擬下載資料（副作用操作）"""
    print(f"    Downloading from {url}...")
    time.sleep(0.5)  # 模擬網路延遲
    return f"Raw data from {url}"


@task
def call_llm_for_analysis(data: str) -> str:
    """模擬 LLM 分析（可能超時的操作）"""
    print(f"    Analyzing data ({len(data)} chars)...")
    time.sleep(0.3)
    return f"Analysis: {data} has been analyzed. Key findings: ..."


# === Node 定義 ===
def fetch_data(state: PipelineState) -> dict:
    """步驟 1：擷取資料"""
    print("  [Node] fetch_data")
    future = download_data(state["source_url"])
    return {"raw_data": future.result()}


def clean_data(state: PipelineState) -> dict:
    """步驟 2：清理資料"""
    print("  [Node] clean_data")
    raw = state["raw_data"]
    cleaned = raw.strip().upper()  # 確定性操作，不需要 task
    return {"cleaned_data": cleaned}


def analyze_data(state: PipelineState) -> dict:
    """步驟 3：分析資料（使用 LLM）"""
    print("  [Node] analyze_data")
    future = call_llm_for_analysis(state["cleaned_data"])
    return {"analysis": future.result()}


def generate_report(state: PipelineState) -> dict:
    """步驟 4：產生報告"""
    print("  [Node] generate_report")
    return {
        "report": (
            f"=== Report ===\n"
            f"Source: {state['source_url']}\n"
            f"Analysis: {state['analysis']}\n"
            f"Generated at: {time.strftime('%Y-%m-%d %H:%M:%S')}"
        )
    }


# === 建構 Graph ===
builder = StateGraph(PipelineState)

builder.add_node("fetch_data", fetch_data)
builder.add_node("clean_data", clean_data)
builder.add_node(
    "analyze_data",
    analyze_data,
    # LLM 呼叫可能暫時失敗，加入重試
    retry_policy=RetryPolicy(max_attempts=3, initial_interval=1.0),
)
builder.add_node("generate_report", generate_report)

builder.add_edge(START, "fetch_data")
builder.add_edge("fetch_data", "clean_data")
builder.add_edge("clean_data", "analyze_data")
builder.add_edge("analyze_data", "generate_report")
builder.add_edge("generate_report", END)

checkpointer = InMemorySaver()
graph = builder.compile(checkpointer=checkpointer)

# === 執行 ===
if __name__ == "__main__":
    thread_id = str(uuid.uuid4())
    config = {"configurable": {"thread_id": thread_id}}

    print("=== 執行 Pipeline ===")
    result = graph.invoke(
        {"source_url": "https://data.example.com/dataset-001"},
        config,
        durability="sync",  # 確保每一步都保存
    )
    print(f"\n最終報告:\n{result['report']}")

    # === 檢視 checkpoint 歷史 ===
    print("\n=== Checkpoint 歷史 ===")
    for state in graph.get_state_history(config):
        step = state.metadata.get("step", "?")
        next_nodes = state.next
        print(f"  Step {step}: next={next_nodes}")
```

## 重點摘要

| 概念 | 重��� |
|------|------|
| **Durable Execution** | 在關鍵點保存進度，暫停後可精確恢復 |
| **啟用方式** | 使用 checkpointer 編譯 graph + 指定 thread_id |
| **確定性** | 副作用和非確定性操作必須放入 `@task` 或 node 中 |
| **恢復機制** | 不是從中斷行繼續，而是從最近 checkpoint 開始重播 |
| **Durability Mode** | `"exit"` / `"async"` / `"sync"` 三種平衡效能與持久性 |
| **@task** | 封裝副作用操作，恢復時從 checkpoint 讀取結果 |
| **Pending Writes** | 平行 node 中已成功的結果會保留，只重新執行失敗的 |
| **故障恢復** | 傳入 `None` 作為 input + 相同 thread_id = 從失敗點恢復 |
