## 1. State Schema 的角色

State Schema 定義了「圖中流動的資料長什麼樣」。它是 LangGraph 應用的**資料合約**——所有節點讀取和寫入都遵循這個合約。

```
                    State Schema
              ┌──────────────────────┐
              │ messages: list[...]  │
              │ current_step: str    │
              │ result: str          │
              │ score: int           │
              └──────────────────────┘
                          │
          ┌───────────────┼───────────────┐
          │               │               │
          v               v               v
     ┌──────────┐    ┌──────────┐    ┌──────────┐
     │  Node A  │    │  Node B  │    │  Node C  │
     │read/write│    │read/write│    │read/write│
     └──────────┘    └──────────┘    └──────────┘
```

**三個核心用途：**
1. **定義型別**：每個欄位的資料型別（int、str、list 等）
2. **定義 Reducer**：每個欄位的更新策略（覆蓋 or 合併）
3. **約束 I/O**：可選地分離輸入/輸出 Schema，控制外部可見的欄位

## 2. 使用 TypedDict 定義 State

### 最常用、效能最好的方式

```python
from typing import TypedDict

class AgentState(TypedDict):
    messages: list[str]
    current_step: str
    result: str
    score: int
```

### 完整範例

```python
"""
使用 TypedDict 定義 State 的完整範例
"""
from typing import TypedDict
from langchain_core.messages import AnyMessage
from langgraph.graph import StateGraph, START, END

# ============================================================
# 1. 定義 State Schema
# ============================================================
class AgentState(TypedDict):
    messages: list[AnyMessage]   # 對話訊息列表
    task: str                    # 目前任務描述
    iteration: int               # 迭代次數
    result: str                  # 最終結果

# ============================================================
# 2. 定義 Nodes
# ============================================================
def initialize(state: AgentState) -> dict:
    """初始化節點：設定初始值"""
    return {
        "task": state.get("task", "未指定"),
        "iteration": 0,
        "result": "",
    }

def process(state: AgentState) -> dict:
    """處理節點：模擬處理邏輯"""
    current_iteration = state["iteration"]
    task = state["task"]
    return {
        "iteration": current_iteration + 1,
        "result": f"已完成任務「{task}」的第 {current_iteration + 1} 次處理",
    }

# ============================================================
# 3. 建構 Graph
# ============================================================
builder = StateGraph(AgentState)
builder.add_node("initialize", initialize)
builder.add_node("process", process)
builder.add_edge(START, "initialize")
builder.add_edge("initialize", "process")
builder.add_edge("process", END)

graph = builder.compile()

# ============================================================
# 4. 執行
# ============================================================
result = graph.invoke({
    "messages": [],
    "task": "分析報告",
    "iteration": 0,
    "result": "",
})

print(result)
# {
#     "messages": [],
#     "task": "分析報告",
#     "iteration": 1,
#     "result": "已完成任務「分析報告」的第 1 次處理"
# }
```

> 📄 完整範例程式碼：[2.1-example-typeddict-state.py](./2.1-example-typeddict-state.py)

### TypedDict 的特性

| 特性 | 說明 |
|------|------|
| **效能** | 最佳——無額外開銷，本質上就是 dict |
| **型別檢查** | 靜態檢查（IDE / mypy），執行時**不驗證** |
| **預設值** | 不支援預設值（需用 `state.get("key", default)`） |
| **巢狀驗證** | 不支援 |

**注意**：TypedDict 在執行時**不做型別檢查**。如果節點回傳了錯誤型別的值，不會立即報錯——直到某個節點嘗試使用該值時才可能出問題。

## 3. 使用 Dataclass 定義 State

### 支援預設值的替代方案

```python
from dataclasses import dataclass, field

@dataclass
class AgentState:
    messages: list[str] = field(default_factory=list)
    task: str = "未指定"
    iteration: int = 0
    result: str = ""
```

### 完整範例

```python
"""
使用 Dataclass 定義 State 的完整範例
展示預設值的便利性
"""
from dataclasses import dataclass, field
from langgraph.graph import StateGraph, START, END

# ============================================================
# 1. 定義 State Schema（有預設值）
# ============================================================
@dataclass
class TaskState:
    task: str = ""                              # 任務描述
    steps: list[str] = field(default_factory=list)  # 已執行步驟
    status: str = "pending"                     # 狀態：pending/running/done
    error_count: int = 0                        # 錯誤計數

# ============================================================
# 2. 定義 Nodes
# ============================================================
def start_task(state: TaskState) -> dict:
    """開始任務"""
    return {
        "status": "running",
        "steps": ["started"],
    }

def execute_task(state: TaskState) -> dict:
    """執行任務"""
    new_steps = state.steps + [f"executed: {state.task}"]
    return {
        "steps": new_steps,
        "status": "done",
    }

# ============================================================
# 3. 建構 Graph
# ============================================================
builder = StateGraph(TaskState)
builder.add_node("start", start_task)
builder.add_node("execute", execute_task)
builder.add_edge(START, "start")
builder.add_edge("start", "execute")
builder.add_edge("execute", END)

graph = builder.compile()

# ============================================================
# 4. 執行——可以只傳部分欄位，其餘用預設值
# ============================================================
result = graph.invoke({"task": "生成報告"})
# 不需要傳 steps、status、error_count，它們有預設值

print(result)
# {
#     "task": "生成報告",
#     "steps": ["started", "executed: 生成報告"],
#     "status": "done",
#     "error_count": 0
# }
```

> 📄 完整範例程式碼：[2.1-example-dataclass-state.py](./2.1-example-dataclass-state.py)

### Dataclass 的特性

| 特性 | 說明 |
|------|------|
| **效能** | 好——比 TypedDict 稍有開銷，但仍然輕量 |
| **型別檢查** | 靜態檢查，執行時**不驗證** |
| **預設值** | ✅ 完整支援（`field(default_factory=...)` 處理 mutable 預設值） |
| **巢狀驗證** | 不支援 |

### 預設值的注意事項

```python
from dataclasses import dataclass, field

@dataclass
class MyState:
    # ✅ 正確：不可變型別可以直接給預設值
    name: str = "default"
    count: int = 0

    # ✅ 正確：可變型別必須用 field(default_factory=...)
    items: list[str] = field(default_factory=list)
    metadata: dict = field(default_factory=dict)

    # ❌ 錯誤：可變型別不能直接給預設值（Python 會報錯）
    # items: list[str] = []     # 這會導致所有實例共用同一個 list！
```

## 4. 使用 Pydantic Model 定義 State

### 提供執行時資料驗證

```python
from pydantic import BaseModel, Field

class AgentState(BaseModel):
    messages: list[str] = []
    task: str = Field(default="", description="任務描述")
    score: int = Field(default=0, ge=0, le=100)  # 0~100 之間
```

### 完整範例

```python
"""
使用 Pydantic Model 定義 State 的完整範例
展示執行時驗證能力
"""
from pydantic import BaseModel, Field
from langgraph.graph import StateGraph, START, END

# ============================================================
# 1. 定義 State Schema（含驗證規則）
# ============================================================
class ReviewState(BaseModel):
    content: str = Field(description="要審核的內容")
    reviewer: str = Field(default="auto", description="審核者名稱")
    score: int = Field(default=0, ge=0, le=100, description="評分 0~100")
    approved: bool = Field(default=False, description="是否通過審核")
    comments: list[str] = Field(default_factory=list, description="審核意見")

# ============================================================
# 2. 定義 Nodes
# ============================================================
def review(state: ReviewState) -> dict:
    """自動審核節點"""
    content = state.content
    # 模擬評分邏輯
    score = min(len(content) * 5, 100)  # 越長分越高（簡化邏輯）
    return {
        "score": score,
        "approved": score >= 60,
        "comments": [f"自動審核完成，評分：{score}"],
    }

def finalize(state: ReviewState) -> dict:
    """完成節點"""
    status = "通過" if state.approved else "未通過"
    return {
        "comments": state.comments + [f"最終結果：{status}"],
    }

# ============================================================
# 3. 建構 Graph
# ============================================================
builder = StateGraph(ReviewState)
builder.add_node("review", review)
builder.add_node("finalize", finalize)
builder.add_edge(START, "review")
builder.add_edge("review", "finalize")
builder.add_edge("finalize", END)

graph = builder.compile()

# ============================================================
# 4. 執行——正常情況
# ============================================================
result = graph.invoke({"content": "這是一篇很棒的技術文章，內容豐富。"})
print(result)
# score: 80, approved: True

# ============================================================
# 5. 執行——驗證錯誤（Pydantic 會攔截）
# ============================================================
try:
    # score 超出範圍 0~100，Pydantic 會拋出 ValidationError
    result = graph.invoke({"content": "test", "score": 999})
except Exception as e:
    print(f"驗證錯誤：{e}")
```

> 📄 完整範例程式碼：[2.1-example-pydantic-state.py](./2.1-example-pydantic-state.py)

### Pydantic 的特性

| 特性 | 說明 |
|------|------|
| **效能** | 較差——遞迴驗證有額外開銷 |
| **型別檢查** | 靜態 + **執行時驗證**（重點功能！） |
| **預設值** | ✅ 完整支援 |
| **巢狀驗證** | ✅ 支援（自動驗證巢狀物件） |

### Pydantic 的已知限制

```
⚠️ 使用 Pydantic 作為 State Schema 的限制：

1. 圖的輸出 不會是 Pydantic model 實例——仍然是 dict
2. 執行時驗證 僅發生在第一個節點的輸入，不會驗證後續節點
3. 驗證錯誤的 stack trace 不會顯示是哪個節點觸發的
4. 效能較差——對效能敏感的應用建議用 dataclass 替代
5. create_agent 高階 API 不支援 Pydantic state schema
```

## 5. 三種定義方式的比較

### 對比表

| 特性 | TypedDict | Dataclass | Pydantic |
|------|-----------|-----------|----------|
| **效能** | ⭐⭐⭐ 最佳 | ⭐⭐ 好 | ⭐ 較慢 |
| **預設值** | ❌ 不支援 | ✅ 支援 | ✅ 支援 |
| **執行時驗證** | ❌ | ❌ | ✅ |
| **IDE 支援** | ✅ 好 | ✅ 好 | ✅ 好 |
| **巢狀驗證** | ❌ | ❌ | ✅ |
| **create_agent 相容** | ✅ | ✅ | ❌ |
| **推薦場景** | 大多數情況 | 需要預設值 | 需要輸入驗證 |

### 選擇指南

```
需要執行時驗證外部輸入？ ──── 是 ──> Pydantic
                         │
                         否
                         │
需要預設值？ ──────────── 是 ──> Dataclass
                         │
                         否
                         │
                         └──────────> TypedDict（預設選擇）
```

### 語法對比

```python
# ===== TypedDict =====
from typing import TypedDict

class State(TypedDict):
    name: str
    items: list[str]
    count: int

# 使用時必須提供所有欄位
graph.invoke({"name": "test", "items": [], "count": 0})


# ===== Dataclass =====
from dataclasses import dataclass, field

@dataclass
class State:
    name: str = ""
    items: list[str] = field(default_factory=list)
    count: int = 0

# 可以只傳部分欄位
graph.invoke({"name": "test"})


# ===== Pydantic =====
from pydantic import BaseModel, Field

class State(BaseModel):
    name: str = ""
    items: list[str] = Field(default_factory=list)
    count: int = Field(default=0, ge=0)  # 額外驗證：>= 0

# 可以只傳部分欄位 + 自動驗證
graph.invoke({"name": "test"})
```

## 6. MessagesState：內建的訊息型 Schema

### 定義

`MessagesState` 是 LangGraph 預建的 State Schema，專為聊天/對話場景設計：

```python
from langgraph.graph import MessagesState

# MessagesState 的定義等同於：
# class MessagesState(TypedDict):
#     messages: Annotated[list[AnyMessage], add_messages]
```

### 為什麼需要 MessagesState？

對話場景是 LLM 應用最常見的模式。`MessagesState` 解決了兩個關鍵問題：

1. **自動追加**：新訊息追加到列表，而非覆蓋
2. **智慧更新**：同 ID 的訊息更新而非重複追加

### 繼承擴展

通常你會繼承 `MessagesState` 並加入自己的欄位：

```python
from langgraph.graph import MessagesState

class MyState(MessagesState):
    """在 messages 基礎上增加自定義欄位"""
    documents: list[str]      # 檢索到的文件
    current_tool: str         # 目前使用的工具
    iteration: int            # 迭代計數
```

### 完整範例

```python
"""
使用 MessagesState 建構聊天機器人
"""
from langgraph.graph import StateGraph, MessagesState, START, END

# ============================================================
# 1. 擴展 MessagesState
# ============================================================
class ChatState(MessagesState):
    user_name: str        # 使用者名稱
    topic: str            # 對話主題

# ============================================================
# 2. 定義 Nodes
# ============================================================
def greet(state: ChatState) -> dict:
    """歡迎節點"""
    name = state.get("user_name", "使用者")
    return {
        "messages": [{"role": "assistant", "content": f"你好，{name}！"}],
        "topic": "greeting",
    }

def respond(state: ChatState) -> dict:
    """回應節點"""
    last_user_msg = state["messages"][-1]
    return {
        "messages": [{"role": "assistant", "content": f"收到你的訊息：{last_user_msg.content}"}],
    }

# ============================================================
# 3. 建構 Graph
# ============================================================
builder = StateGraph(ChatState)
builder.add_node("greet", greet)
builder.add_node("respond", respond)
builder.add_edge(START, "greet")
builder.add_edge("greet", "respond")
builder.add_edge("respond", END)

graph = builder.compile()

# ============================================================
# 4. 執行
# ============================================================
result = graph.invoke({
    "messages": [{"role": "user", "content": "請幫我分析資料"}],
    "user_name": "小明",
    "topic": "",
})

for msg in result["messages"]:
    print(f"{msg.type}: {msg.content}")
# human: 請幫我分析資料
# ai: 你好，小明！
# ai: 收到你的訊息：請幫我分析資料
```

> 📄 完整範例程式碼：[2.1-example-messages-state.py](./2.1-example-messages-state.py)

### 訊息格式支援

`add_messages` reducer 支援自動反序列化，以下格式都可以：

```python
# 格式一：OpenAI 風格的 dict
{"role": "user", "content": "你好"}
{"role": "assistant", "content": "你好！"}

# 格式二：LangChain Message 物件
from langchain_core.messages import HumanMessage, AIMessage
HumanMessage(content="你好")
AIMessage(content="你好！")

# 格式三：簡化的 tuple
("user", "你好")
("assistant", "你好！")
```

## 7. Input / Output Schema 分離設計

### 為什麼要分離？

在實際應用中，State 往往包含許多內部欄位。你不希望：
- 外部呼叫者需要提供所有欄位
- 回傳結果包含內部處理的中間值

### 完整範例

```python
"""
Input / Output Schema 分離設計
展示如何控制圖的輸入和輸出介面
"""
from typing import TypedDict
from langgraph.graph import StateGraph, START, END

# ============================================================
# 1. 定義三層 Schema
# ============================================================

# 外部輸入只需要這些
class InputState(TypedDict):
    question: str               # 使用者問題

# 外部輸出只回傳這些
class OutputState(TypedDict):
    answer: str                 # 最終答案
    confidence: float           # 信心分數

# 內部完整 State（包含所有欄位）
class OverallState(InputState, OutputState):
    # 繼承自 InputState: question
    # 繼承自 OutputState: answer, confidence
    retrieved_docs: list[str]   # 內部用：檢索到的文件
    reasoning: str              # 內部用：推理過程
    attempt_count: int          # 內部用：嘗試次數

# ============================================================
# 2. 定義 Nodes
# ============================================================
def retrieve(state: InputState) -> dict:
    """檢索節點：根據問題搜尋相關文件"""
    question = state["question"]
    # 模擬文件檢索
    docs = [
        f"文件1：關於 {question} 的背景資料...",
        f"文件2：{question} 的最新研究...",
    ]
    return {
        "retrieved_docs": docs,
        "attempt_count": 1,
    }

def reason(state: OverallState) -> dict:
    """推理節點：根據檢索結果進行推理"""
    docs = state["retrieved_docs"]
    question = state["question"]
    reasoning = f"基於 {len(docs)} 篇文件，分析問題「{question}」..."
    return {"reasoning": reasoning}

def answer(state: OverallState) -> dict:
    """回答節點：生成最終答案"""
    return {
        "answer": f"根據分析：{state['reasoning']}，結論是...",
        "confidence": 0.85,
    }

# ============================================================
# 3. 建構 Graph（指定 input/output schema）
# ============================================================
builder = StateGraph(
    OverallState,                  # 內部完整 Schema
    input_schema=InputState,       # 外部輸入限制
    output_schema=OutputState,     # 外部輸出限制
)

builder.add_node("retrieve", retrieve)
builder.add_node("reason", reason)
builder.add_node("answer", answer)

builder.add_edge(START, "retrieve")
builder.add_edge("retrieve", "reason")
builder.add_edge("reason", "answer")
builder.add_edge("answer", END)

graph = builder.compile()

# ============================================================
# 4. 執行
# ============================================================

# 輸入：只需傳 InputState 的欄位
result = graph.invoke({"question": "LangGraph 是什麼？"})

print(result)
# 輸出只包含 OutputState 的欄位：
# {"answer": "根據分析：...，結論是...", "confidence": 0.85}
#
# 注意：retrieved_docs、reasoning、attempt_count 不會出現在輸出中！
```

> 📄 完整範例程式碼：[2.1-example-input-output-schema.py](./2.1-example-input-output-schema.py)

### 運作原理圖

```
外部輸入                 內部 State                        外部輸出
┌──────────┐     ┌──────────────────────────┐     ┌───────────────┐
│ question │────>│ question                 │     │ answer        │
└──────────┘     │ answer                   │────>│ confidence    │
                 │ confidence               │     └───────────────┘
InputState       │ retrieved_docs  (內部用)  │     OutputState
                 │ reasoning       (內部用)  │
                 │ attempt_count   (內部用)  │
                 └──────────────────────────┘
                       OverallState
```

### 關鍵規則

1. `OverallState` 必須包含 `InputState` 和 `OutputState` 的所有欄位
2. **節點可以寫入 OverallState 中的任何欄位**——即使該欄位不在節點的輸入 Schema 中
3. **節點的參數型別標註**決定它讀取哪些欄位，但不限制它寫入哪些欄位
4. `invoke()` 的輸入會被 `InputState` 過濾
5. `invoke()` 的輸出會被 `OutputState` 過濾

## 8. State Schema 設計原則

### 原則一：從最小開始

```python
# ✅ 好：只包含必要欄位
class State(TypedDict):
    messages: Annotated[list[AnyMessage], add_messages]
    result: str

# ❌ 差：預先定義一堆「可能會用到」的欄位
class State(TypedDict):
    messages: Annotated[list[AnyMessage], add_messages]
    result: str
    temp_data: dict          # 還不確定用途
    debug_info: str          # 只是「以防萬一」
    extra_field_1: str       # 備用
    extra_field_2: str       # 備用
```

### 原則二：善用 Reducer 避免手動合併

```python
# ❌ 差：手動合併列表
def node(state):
    existing = state["logs"]
    return {"logs": existing + ["new log"]}

# ✅ 好：用 Reducer 自動合併
class State(TypedDict):
    logs: Annotated[list[str], operator.add]

def node(state):
    return {"logs": ["new log"]}  # Reducer 自動追加
```

### 原則三：分離關注點

```python
# ✅ 好：Input/Output Schema 分離
class InputState(TypedDict):
    query: str

class OutputState(TypedDict):
    answer: str

class InternalState(InputState, OutputState):
    intermediate_results: list[str]
    processing_stage: str
```

### 原則四：命名要有意義

```python
# ❌ 差
class State(TypedDict):
    data: dict
    flag: bool
    items: list

# ✅ 好
class State(TypedDict):
    search_results: dict
    requires_approval: bool
    retrieved_documents: list[str]
```

## 重點摘要

| 問題 | 答案 |
|------|------|
| 預設該用哪種定義方式？ | `TypedDict`（效能最好，最常用） |
| 需要預設值？ | `dataclass` |
| 需要執行時驗證？ | `Pydantic BaseModel` |
| 最常用的內建 Schema？ | `MessagesState`（對話場景） |
| 怎麼控制外部輸入/輸出？ | `StateGraph(OverallState, input_schema=..., output_schema=...)` |
| 節點需要回傳完整 State 嗎？ | 不需要，只回傳要更新的欄位 |
