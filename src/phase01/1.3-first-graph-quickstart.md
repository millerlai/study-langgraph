## 1. 學習路線圖

本節透過**三個逐步增強的聊天機器人**，帶你從零開始實際建構 LangGraph 應用：

```
範例一：基本聊天機器人          範例二：加入工具            範例三：加入記憶
┌─────────────────┐        ┌─────────────────┐       ┌─────────────────┐
│ Pure LLM Chat   │  ───>  │ LLM+Invoke Tool │  ───> │   LLM+工具+記憶  │
```

## 3. 範例一：基本聊天機器人

### 目標

建立一個最簡單的聊天機器人——接收使用者訊息，呼叫 LLM，回傳回應。沒有工具、沒有記憶，純粹展示 LangGraph 的基本結構。

### 完整程式碼

```python
"""
範例一：基本聊天機器人
- 使用 StateGraph + MessagesState
- 單一 LLM 節點
- 最簡單的 START → llm → END 線性流程
"""
from langchain_openai import ChatOpenAI
from langgraph.graph import StateGraph, MessagesState, START, END

# ============================================================
# 1. 初始化 LLM
# ============================================================
model = ChatOpenAI(model="gpt-4o-mini", temperature=0)

# ============================================================
# 2. 定義 Node
# ============================================================
def chatbot(state: MessagesState) -> dict:
    """
    聊天機器人節點：
    - 讀取 state["messages"]（完整對話歷史）
    - 呼叫 LLM
    - 回傳新的 AI 訊息
    """
    response = model.invoke(state["messages"])
    # 回傳 dict，LangGraph 會用 add_messages reducer
    # 將新訊息追加到 messages 列表
    return {"messages": [response]}

# ============================================================
# 3. 建構 Graph
# ============================================================
builder = StateGraph(MessagesState)

# 註冊節點
builder.add_node("chatbot", chatbot)

# 定義邊：START → chatbot → END
builder.add_edge(START, "chatbot")
builder.add_edge("chatbot", END)

# 編譯成可執行的 Graph
graph = builder.compile()

# ============================================================
# 4. 執行
# ============================================================
result = graph.invoke({
    "messages": [{"role": "user", "content": "你好！請用一句話介紹 LangGraph。"}]
})

# 印出 AI 的回應
print(result["messages"][-1].content)
```

> 📄 完整範例程式碼：[1.3-example-basic-chatbot.py](./1.3-example-basic-chatbot.py)

### 圖結構

```
START ──> chatbot ──> END
```

這是最簡單的線性圖：只有一個節點，一進一出。

### 關鍵概念解說

#### MessagesState

```python
from langgraph.graph import MessagesState

# MessagesState 等同於：
# class MessagesState(TypedDict):
#     messages: Annotated[list[AnyMessage], add_messages]
```

- 內建的 State Schema，專為對話場景設計
- 只有一個欄位 `messages`，型別是訊息列表
- 使用 `add_messages` reducer——新訊息**追加**到列表，同 ID 訊息會**更新**

#### 訊息格式

LangGraph 支援兩種訊息格式：

```python
# 格式一：字典（簡潔，推薦給初學者）
{"role": "user", "content": "你好"}
{"role": "assistant", "content": "你好！"}

# 格式二：LangChain Message 物件（功能更豐富）
from langchain_core.messages import HumanMessage, AIMessage
HumanMessage(content="你好")
AIMessage(content="你好！")
```

`add_messages` reducer 會自動反序列化——不管你傳哪種格式，內部都會轉成 LangChain Message 物件。

#### invoke 的輸入與輸出

```python
# 輸入：符合 MessagesState 的 dict
input_data = {"messages": [{"role": "user", "content": "你好"}]}

# 輸出：完整的 State（包含所有訊息）
result = graph.invoke(input_data)
# result = {
#     "messages": [
#         HumanMessage(content="你好"),        ← 你的輸入
#         AIMessage(content="你好！有什麼..."), ← LLM 的回應
#     ]
# }

# 取得最後一則訊息（AI 回應）
print(result["messages"][-1].content)
```

### 視覺化

```python
# 輸出 Mermaid 格式的圖表
print(graph.get_graph().draw_mermaid())

# 儲存為 PNG 圖片
graph.get_graph().draw_mermaid_png(output_file_path="chatbot_v1.png")
```

## 4. 範例二：加入工具（Tool Calling）

### 目標

讓聊天機器人能呼叫外部工具——LLM 判斷何時需要工具、呼叫哪個工具、傳什麼參數，然後觀察工具結果，最終生成回應。這就是經典的 **ReAct 模式**（Reasoning + Acting）。

### 完整程式碼

```python
"""
範例二：加入工具（Tool Calling）
- LLM 可以決定是否呼叫工具
- 使用 ToolNode 自動執行工具
- 使用 tools_condition 條件路由
- 形成 ReAct 迴圈：llm → tool → llm → ... → END
"""
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from langgraph.graph import StateGraph, MessagesState, START, END
from langgraph.prebuilt import ToolNode, tools_condition

# ============================================================
# 1. 定義工具
# ============================================================
@tool
def get_weather(city: str) -> str:
    """查詢指定城市的天氣資訊。

    Args:
        city: 要查詢天氣的城市名稱
    """
    # 模擬天氣 API（實際應用中會呼叫真正的 API）
    weather_data = {
        "台北": "☀️ 晴天，28°C，濕度 65%",
        "東京": "🌤️ 多雲，22°C，濕度 55%",
        "紐約": "🌧️ 小雨，15°C，濕度 80%",
    }
    return weather_data.get(city, f"抱歉，找不到 {city} 的天氣資料")

@tool
def calculate(expression: str) -> str:
    """計算數學運算式。

    Args:
        expression: 要計算的數學運算式，例如 "2 + 3 * 4"
    """
    try:
        result = eval(expression)  # 注意：正式環境請用安全的計算方式
        return f"計算結果：{expression} = {result}"
    except Exception as e:
        return f"計算錯誤：{e}"

# 工具列表
tools = [get_weather, calculate]

# ============================================================
# 2. 初始化 LLM 並綁定工具
# ============================================================
model = ChatOpenAI(model="gpt-4o-mini", temperature=0)

# bind_tools 讓 LLM 知道有哪些工具可用
# LLM 會在適當時機自動決定是否呼叫工具
model_with_tools = model.bind_tools(tools)

# ============================================================
# 3. 定義 Nodes
# ============================================================
def chatbot(state: MessagesState) -> dict:
    """
    LLM 節點：
    - 使用綁定了工具的 model
    - LLM 可能回傳純文字回應（不需要工具）
    - 也可能回傳 tool_calls（需要呼叫工具）
    """
    response = model_with_tools.invoke(state["messages"])
    return {"messages": [response]}

# ToolNode 是 LangGraph 的預建節點
# 它會自動：
#   1. 讀取最後一則 AIMessage 中的 tool_calls
#   2. 執行對應的工具函數
#   3. 將結果包裝成 ToolMessage 回傳
tool_node = ToolNode(tools)

# ============================================================
# 4. 建構 Graph
# ============================================================
builder = StateGraph(MessagesState)

# 註冊節點
builder.add_node("chatbot", chatbot)
builder.add_node("tools", tool_node)

# 定義邊
builder.add_edge(START, "chatbot")

# 條件邊：chatbot 完成後，檢查是否有 tool_calls
# tools_condition 是 LangGraph 預建的路由函數：
#   - 如果 AIMessage 有 tool_calls → 導向 "tools" 節點
#   - 如果沒有 tool_calls → 導向 END
builder.add_conditional_edges("chatbot", tools_condition)

# 工具執行完後 → 回到 chatbot（讓 LLM 看到工具結果）
builder.add_edge("tools", "chatbot")

# 編譯
graph = builder.compile()

# ============================================================
# 5. 執行——不需要工具的情況
# ============================================================
print("=== 測試一：不需要工具 ===")
result = graph.invoke({
    "messages": [{"role": "user", "content": "你好！你是誰？"}]
})
print(result["messages"][-1].content)

# ============================================================
# 6. 執行——需要工具的情況
# ============================================================
print("\n=== 測試二：查天氣（需要工具） ===")
result = graph.invoke({
    "messages": [{"role": "user", "content": "台北現在天氣如何？"}]
})
print(result["messages"][-1].content)

# ============================================================
# 7. 執行——需要多個工具的情況
# ============================================================
print("\n=== 測試三：複合問題（可能用多個工具） ===")
result = graph.invoke({
    "messages": [{"role": "user", "content": "台北和東京的溫度差幾度？請幫我算一下。"}]
})
print(result["messages"][-1].content)
```

### 關鍵概念解說

#### @tool 裝飾器

```python
from langchain_core.tools import tool

@tool
def my_tool(param: str) -> str:
    """工具的描述——LLM 用這段描述來判斷何時使用此工具。

    Args:
        param: 參數說明——LLM 用這段來決定傳什麼值
    """
    return "result"
```

**重要**：
- 函數的 **docstring** 會成為工具描述，LLM 靠它判斷何時呼叫
- 參數的 **type hints** 定義工具的輸入 schema
- 名稱建議用 **snake_case**（某些 LLM 供應商不支援空格或特殊字元）

#### bind_tools：讓 LLM 知道可用工具

```python
model_with_tools = model.bind_tools(tools)
```

`bind_tools` 做了什麼？
1. 將每個工具的名稱、描述、參數 schema 轉換為 LLM 理解的格式
2. 附加到每次 LLM 呼叫的請求中
3. LLM 在回應時**可以選擇**是否呼叫工具

LLM 的回應有兩種可能：

```python
# 可能一：純文字回應（不需要工具）
AIMessage(content="你好！我是 AI 助手。")

# 可能二：工具呼叫（LLM 決定需要工具）
AIMessage(
    content="",
    tool_calls=[
        {"name": "get_weather", "args": {"city": "台北"}, "id": "call_abc123"}
    ]
)
```

#### ToolNode：自動執行工具

```python
from langgraph.prebuilt import ToolNode

tool_node = ToolNode(tools)
```

`ToolNode` 是 LangGraph 預建的節點，自動處理：

1. 從最後一則 `AIMessage` 取出 `tool_calls`
2. 找到對應的工具函數並執行
3. 支援**平行執行**多個工具呼叫
4. 將結果包裝成 `ToolMessage` 回傳

```
AIMessage(tool_calls=[{name: "get_weather", args: {city: "台北"}}])
    │
    v  ToolNode 自動處理
    │
ToolMessage(content="☀️ 晴天，28°C，濕度 65%", tool_call_id="call_abc123")
```

#### tools_condition：條件路由

```python
from langgraph.prebuilt import tools_condition

builder.add_conditional_edges("chatbot", tools_condition)
```

`tools_condition` 是 LangGraph 預建的路由函數，邏輯等同於：

```python
def tools_condition(state: MessagesState) -> str:
    last_message = state["messages"][-1]
    if hasattr(last_message, "tool_calls") and last_message.tool_calls:
        return "tools"    # 導向 tools 節點
    return END            # 導向 END（結束）
```

#### ReAct 迴圈的完整流程

```
使用者："台北和東京的溫度差幾度？"
    │
    v
[chatbot] LLM 判斷需要查天氣
    │ tool_calls: [get_weather("台北"), get_weather("東京")]
    v
[tools] 平行執行兩個工具
    │ ToolMessage: "☀️ 晴天，28°C..."
    │ ToolMessage: "🌤️ 多雲，22°C..."
    v
[chatbot] LLM 看到工具結果，可能需要計算
    │ tool_calls: [calculate("28 - 22")]
    v
[tools] 執行計算
    │ ToolMessage: "計算結果：28 - 22 = 6"
    v
[chatbot] LLM 整合所有結果，生成最終回應
    │ content: "台北 28°C，東京 22°C，溫度差 6°C。"
    v
[END] 完成
```

**迴圈次數不固定**——LLM 自己決定何時已經收集到足夠資訊可以回答。

#### 自己實作 tool_node（不用 ToolNode 預建元件）

如果你想更深入理解，以下是不使用 `ToolNode` 的手動實作：

```python
import json
from langchain_core.messages import ToolMessage

# 建立工具名稱到函數的映射
tools_by_name = {t.name: t for t in tools}

def tool_node(state: MessagesState) -> dict:
    """手動實作的工具執行節點"""
    result_messages = []
    # 取得最後一則 AI 訊息中的所有 tool_calls
    for tool_call in state["messages"][-1].tool_calls:
        # 找到對應的工具函數
        tool_func = tools_by_name[tool_call["name"]]
        # 執行工具
        observation = tool_func.invoke(tool_call["args"])
        # 包裝成 ToolMessage
        result_messages.append(
            ToolMessage(
                content=observation,
                tool_call_id=tool_call["id"],  # 必須對應 tool_call 的 id
            )
        )
    return {"messages": result_messages}
```

同樣地，`tools_condition` 也可以手動實作：

```python
def should_continue(state: MessagesState) -> str:
    """手動實作的條件路由"""
    last_message = state["messages"][-1]
    if last_message.tool_calls:
        return "tools"
    return END

builder.add_conditional_edges("chatbot", should_continue, ["tools", END])
```

## 5. 範例三：加入記憶（Checkpointing）

### 目標

讓聊天機器人擁有**對話記憶**——記住之前說過的話，支援多輪對話。這是透過 LangGraph 的 Checkpointer 機制實現的。

### 完整程式碼

```python
"""
範例三：加入記憶（Checkpointing）
- 在範例二的基礎上加入 Checkpointer
- 支援多輪對話（記住之前說過的話）
- 使用 thread_id 區分不同的對話 session
- 示範跨輪次的上下文保持
"""
from langchain_openai import ChatOpenAI
from langchain_core.tools import tool
from langgraph.graph import StateGraph, MessagesState, START, END
from langgraph.prebuilt import ToolNode, tools_condition
from langgraph.checkpoint.memory import InMemorySaver

# ============================================================
# 1. 定義工具（同範例二）
# ============================================================
@tool
def get_weather(city: str) -> str:
    """查詢指定城市的天氣資訊。

    Args:
        city: 要查詢天氣的城市名稱
    """
    weather_data = {
        "台北": "☀️ 晴天，28°C，濕度 65%",
        "東京": "🌤️ 多雲，22°C，濕度 55%",
        "紐約": "🌧️ 小雨，15°C，濕度 80%",
    }
    return weather_data.get(city, f"抱歉，找不到 {city} 的天氣資料")

@tool
def calculate(expression: str) -> str:
    """計算數學運算式。

    Args:
        expression: 要計算的數學運算式，例如 "2 + 3 * 4"
    """
    try:
        result = eval(expression)
        return f"計算結果：{expression} = {result}"
    except Exception as e:
        return f"計算錯誤：{e}"

tools = [get_weather, calculate]

# ============================================================
# 2. 初始化 LLM
# ============================================================
model = ChatOpenAI(model="gpt-4o-mini", temperature=0)
model_with_tools = model.bind_tools(tools)

# ============================================================
# 3. 定義 Nodes（同範例二）
# ============================================================
def chatbot(state: MessagesState) -> dict:
    """LLM 節點"""
    response = model_with_tools.invoke(state["messages"])
    return {"messages": [response]}

tool_node = ToolNode(tools)

# ============================================================
# 4. 建構 Graph（同範例二）
# ============================================================
builder = StateGraph(MessagesState)

builder.add_node("chatbot", chatbot)
builder.add_node("tools", tool_node)

builder.add_edge(START, "chatbot")
builder.add_conditional_edges("chatbot", tools_condition)
builder.add_edge("tools", "chatbot")

# ============================================================
# 5. 關鍵差異：使用 Checkpointer 編譯
# ============================================================
# InMemorySaver 將 State 保存在記憶體中（開發用）
# 正式環境可用 SqliteSaver 或 PostgresSaver
checkpointer = InMemorySaver()

graph = builder.compile(checkpointer=checkpointer)
#                       ^^^^^^^^^^^^^^^^^^^^^^^^
#                       唯一的差異：加了 checkpointer！

# ============================================================
# 6. 多輪對話示範
# ============================================================

# thread_id 是對話的唯一識別碼
# 同一個 thread_id = 同一段對話，會保留上下文
config = {"configurable": {"thread_id": "conversation-001"}}

# --- 第一輪 ---
print("=== 第一輪 ===")
result = graph.invoke(
    {"messages": [{"role": "user", "content": "你好！我叫小明。"}]},
    config,  # 傳入 config 指定 thread_id
)
print(f"AI: {result['messages'][-1].content}")

# --- 第二輪（AI 應該記得你叫小明）---
print("\n=== 第二輪 ===")
result = graph.invoke(
    {"messages": [{"role": "user", "content": "你還記得我叫什麼名字嗎？"}]},
    config,  # 同一個 thread_id → 接續同一段對話
)
print(f"AI: {result['messages'][-1].content}")

# --- 第三輪（使用工具 + 記憶）---
print("\n=== 第三輪 ===")
result = graph.invoke(
    {"messages": [{"role": "user", "content": "台北天氣如何？"}]},
    config,
)
print(f"AI: {result['messages'][-1].content}")

# --- 第四輪（AI 應該記得之前查過天氣）---
print("\n=== 第四輪 ===")
result = graph.invoke(
    {"messages": [{"role": "user", "content": "剛剛查到的溫度是幾度？"}]},
    config,
)
print(f"AI: {result['messages'][-1].content}")

# ============================================================
# 7. 不同的 thread_id = 全新的對話
# ============================================================
print("\n=== 全新對話（不同 thread_id） ===")
new_config = {"configurable": {"thread_id": "conversation-002"}}
result = graph.invoke(
    {"messages": [{"role": "user", "content": "你還記得我叫什麼嗎？"}]},
    new_config,  # 不同的 thread_id → 全新對話，沒有記憶
)
print(f"AI: {result['messages'][-1].content}")
# AI 不會記得「小明」，因為這是不同的 thread
```

### 關鍵概念解說

#### Checkpointer 是什麼？

Checkpointer 是 LangGraph 的**持久化層**。啟用後，每個 super-step 結束時都會自動保存一份 State 快照（checkpoint）。

```
                        Checkpointer 的運作
                        
invoke({messages: ["你好"]}, config={thread_id: "001"})
    │
    v
 [START]  ── checkpoint #0：空 State
    │
    v
 [chatbot] ── checkpoint #1：包含 user + AI 訊息
    │
    v
  [END]

invoke({messages: ["台北天氣？"]}, config={thread_id: "001"})
    │
    v                               ┌─────────────────────────┐
 [START]  ← 自動載入 checkpoint #1   │ 此時 state["messages"]  │
    │        的 State               │ 已經包含第一輪的          │
    v                               │ "你好" + AI 回應         │
 [chatbot] ── checkpoint #2         │ + 新的 "台北天氣？"      │
    │                               └─────────────────────────┘
    v
 [tools]  ── checkpoint #3
    │
    v
 [chatbot] ── checkpoint #4
    │
    v
  [END]
```

#### Thread（對話線程）

`thread_id` 是對話的唯一識別碼。不同 `thread_id` 代表完全獨立的對話：

```python
# 對話 A
config_a = {"configurable": {"thread_id": "alice-session"}}
graph.invoke({"messages": [...]}, config_a)

# 對話 B（完全獨立，不共享記憶）
config_b = {"configurable": {"thread_id": "bob-session"}}
graph.invoke({"messages": [...]}, config_b)
```

```
Thread "alice-session"              Thread "bob-session"
┌─────────────────────┐           ┌─────────────────────┐
│ checkpoint #0       │           │ checkpoint #0       │
│ checkpoint #1       │           │ checkpoint #1       │
│ checkpoint #2       │           │ ...                 │
│ ...                 │           │                     │
└─────────────────────┘           └─────────────────────┘
  完全獨立，互不影響                 完全獨立，互不影響
```

#### Checkpointer 的種類

| Checkpointer | 適用場景 | 安裝 |
|--------------|---------|------|
| **`InMemorySaver`** | 開發、測試（程式結束即遺失） | 內建，無需額外安裝 |
| **`SqliteSaver`** | 本地開發、單機部署 | `uv add langgraph-checkpoint-sqlite` |
| **`PostgresSaver`** | 正式環境、多實例部署 | `uv add langgraph-checkpoint-postgres` |

```python
# 開發環境
from langgraph.checkpoint.memory import InMemorySaver
checkpointer = InMemorySaver()

# 本地持久化
import sqlite3
from langgraph.checkpoint.sqlite import SqliteSaver
conn = sqlite3.connect("checkpoints.db")
checkpointer = SqliteSaver(conn)

# 正式環境
from langgraph.checkpoint.postgres import PostgresSaver
checkpointer = PostgresSaver.from_conn_string("postgresql://user:pass@host/db")
checkpointer.setup()  # 建立必要的資料表
```

#### 查看 State 與歷史

Checkpointer 不只提供記憶，還可以查看和操作 State：

```python
config = {"configurable": {"thread_id": "conversation-001"}}

# 查看目前 State
snapshot = graph.get_state(config)
print(snapshot.values)   # 目前的 State 值
print(snapshot.next)     # 下一個要執行的節點（如果還在執行中）

# 查看完整歷史
for state in graph.get_state_history(config):
    print(f"Step {state.metadata['step']}: {state.next}")
    print(f"  Messages: {len(state.values.get('messages', []))}")
```

#### 為什麼 invoke 只需傳「新訊息」？

注意範例中每輪對話只傳新的使用者訊息：

```python
# 第一輪
graph.invoke({"messages": [{"role": "user", "content": "你好"}]}, config)

# 第二輪——只傳新訊息，不需要重傳第一輪的內容
graph.invoke({"messages": [{"role": "user", "content": "你記得我嗎？"}]}, config)
```

這是因為：
1. Checkpointer 已經保存了之前的完整 State（包含所有舊訊息）
2. 新的輸入會透過 `add_messages` reducer **追加**到已保存的訊息列表
3. LLM 看到的是**完整的對話歷史**（舊訊息 + 新訊息）

```
Checkpointer 中的 State:     你傳入的新訊息:        LLM 看到的:
[User: "你好"]          +   [User: "你記得我嗎？"]  =  [User: "你好"]
[AI: "你好！我是..."]                                  [AI: "你好！我是..."]
                                                       [User: "你記得我嗎？"]
```

## 6. 三個版本的演進對比

### 程式碼差異

```python
# ===== 範例一 → 範例二的差異 =====

# 新增：定義工具
@tool
def get_weather(city: str) -> str: ...

# 新增：綁定工具到 LLM
model_with_tools = model.bind_tools(tools)

# 變更：Node 中使用 model_with_tools
def chatbot(state):
    response = model_with_tools.invoke(state["messages"])  # 改用 model_with_tools
    return {"messages": [response]}

# 新增：ToolNode
tool_node = ToolNode(tools)
builder.add_node("tools", tool_node)

# 變更：邊——從固定邊改為條件邊 + 迴圈
builder.add_conditional_edges("chatbot", tools_condition)  # 取代 add_edge(chatbot, END)
builder.add_edge("tools", "chatbot")                       # 新增迴圈邊


# ===== 範例二 → 範例三的差異 =====

# 新增：Checkpointer
checkpointer = InMemorySaver()
graph = builder.compile(checkpointer=checkpointer)  # 只多了這個參數

# 變更：invoke 時傳入 config
config = {"configurable": {"thread_id": "001"}}
graph.invoke(input, config)                          # 多了 config 參數
```

## 重點摘要

| 問題 | 答案 |
|------|------|
| 最簡單的 Graph 需要什麼？ | `StateGraph` + 至少一個 Node + `START`/`END` 邊 + `compile()` |
| 怎麼讓 LLM 呼叫工具？ | `@tool` 定義 + `bind_tools` 綁定 + `ToolNode` 執行 + `tools_condition` 路由 |
| 怎麼讓機器人有記憶？ | `compile(checkpointer=...)` + `invoke` 時傳 `thread_id` |
| Thread 是什麼？ | 一個 `thread_id` = 一段獨立對話，有自己的 State 歷史 |
| Checkpoint 是什麼？ | 每個 super-step 後的 State 快照，支援恢復與時間旅行 |
| 為什麼要用 `tools_condition`？ | 它自動判斷 LLM 回應是否包含 `tool_calls` 來決定路由 |
| 開發用哪種 Checkpointer？ | `InMemorySaver`（記憶體，重啟即遺失） |
| 正式環境用哪種？ | `PostgresSaver`（持久化到資料庫） |
