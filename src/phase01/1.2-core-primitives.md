## 1. 總覽：八大核心原語

LangGraph 的所有功能都建立在一小組核心原語之上。理解這些原語就等於理解了 LangGraph 的根本運作方式。

```
┌─────────────────────────────────────────────────────────────────┐
│                    LangGraph 核心原語總覽                        │
├──────────────┬──────────────────────────────────────────────────┤
│ StateGraph   │ 圖的容器——定義 State Schema，承載所有 Node 與 Edge │
├──────────────┼──────────────────────────────────────────────────┤
│ State        │ 共享資料結構——節點之間的「記憶」                    │
├──────────────┼──────────────────────────────────────────────────┤
│ Channel      │ State 中的每個欄位，各自獨立管理更新                │
├──────────────┼──────────────────────────────────────────────────┤
│ Reducer      │ 定義 Channel 收到更新時的合併策略                  │
├──────────────┼──────────────────────────────────────────────────┤
│ Node         │ 工作單元——一個 Python 函數                        │
├──────────────┼──────────────────────────────────────────────────┤
│ Edge         │ 節點之間的連接——決定執行順序                       │
├──────────────┼──────────────────────────────────────────────────┤
│ START / END  │ 虛擬節點——標記圖的入口與出口                       │
├──────────────┼──────────────────────────────────────────────────┤
│ Command      │ 進階原語——在節點內同時更新 State 與控制流           │
└──────────────┴──────────────────────────────────────────────────┘
```

## 2. StateGraph：主要的圖抽象

### 定義

`StateGraph` 是 LangGraph 的**圖容器**（Graph Container）。它是你建構任何 LangGraph 應用的起點。你用它來：

1. 定義 State Schema（圖中流動的資料長什麼樣）
2. 註冊 Node（有哪些工作單元）
3. 宣告 Edge（節點之間怎麼連接）
4. 編譯成可執行的 Graph

### 基本用法

```python
from langgraph.graph import StateGraph, START, END
from typing import TypedDict

# 1. 定義 State Schema
class MyState(TypedDict):
    counter: int
    message: str

# 2. 建立 StateGraph，傳入 State Schema
builder = StateGraph(MyState)

# 3. 加入節點
builder.add_node("step_a", func_a)
builder.add_node("step_b", func_b)

# 4. 加入邊
builder.add_edge(START, "step_a")
builder.add_edge("step_a", "step_b")
builder.add_edge("step_b", END)

# 5. 編譯
graph = builder.compile()

# 6. 執行
result = graph.invoke({"counter": 0, "message": ""})
```

### 編譯（Compile）

`builder.compile()` 做了什麼？

1. **結構驗證**：檢查是否有孤立節點（沒有任何邊連接的節點）
2. **入口檢查**：確認有從 `START` 出發的邊
3. **設定 Runtime 參數**：可在此指定 `checkpointer`、`breakpoints` 等

```python
from langgraph.checkpoint.memory import InMemorySaver

# 編譯時可指定選項
graph = builder.compile(
    checkpointer=InMemorySaver(),   # 啟用持久化
    interrupt_before=["sensitive"],  # 在指定節點前暫停（靜態 breakpoint）
)
```

**重要**：你**必須**先 compile 才能使用 graph。`builder` 只是建構器，`graph` 才是可執行物件。

### StateGraph 的 Schema 參數

`StateGraph` 支援多種 Schema 設定：

```python
# 基本用法：單一 Schema
builder = StateGraph(MyState)

# 進階用法：分離 Input / Output Schema
builder = StateGraph(
    OverallState,                # 內部完整 Schema
    input_schema=InputState,     # 限制外部輸入只能傳這些欄位
    output_schema=OutputState,   # 限制外部輸出只回傳這些欄位
)

# 進階用法：Runtime Context（執行時注入依賴）
builder = StateGraph(
    MyState,
    context_schema=ContextSchema,  # 透過 context 參數傳入
)
```

## 3. State：節點之間傳遞的核心資料結構

### 定義

State 是整個 Graph 中的**共享記憶體**——所有節點都讀取它、更新它。State 定義了「圖中流動的資料長什麼樣」。

### 三種定義方式

#### 方式一：TypedDict（最常用、效能最好）

```python
from typing import TypedDict

class AgentState(TypedDict):
    messages: list[str]
    current_step: str
    result: str
```

#### 方式二：Dataclass（支援預設值）

```python
from dataclasses import dataclass, field

@dataclass
class AgentState:
    messages: list[str] = field(default_factory=list)
    current_step: str = "init"
    result: str = ""
```

#### 方式三：Pydantic BaseModel（支援資料驗證，但效能較差）

```python
from pydantic import BaseModel

class AgentState(BaseModel):
    messages: list[str] = []
    current_step: str = "init"
    result: str = ""
```

### State 的更新方式

**關鍵概念**：節點不需要回傳完整的 State，只需要回傳**要更新的欄位**。

```python
class MyState(TypedDict):
    foo: int
    bar: str
    baz: list[str]

def my_node(state: MyState) -> dict:
    # 只更新 foo，bar 和 baz 維持原值
    return {"foo": state["foo"] + 1}
```

更新流程：

```
初始 State:   {"foo": 1, "bar": "hello", "baz": ["a"]}
                                │
                    Node 回傳: {"foo": 2}
                                │
                                v
更新後 State: {"foo": 2, "bar": "hello", "baz": ["a"]}
              ↑ 更新      ↑ 不變         ↑ 不變
```

### Input / Output Schema 分離

當你的 State 有很多內部欄位，但不想全部暴露給外部呼叫者時：

```python
class InputState(TypedDict):
    user_input: str              # 外部只需傳入這個

class OutputState(TypedDict):
    graph_output: str            # 外部只會拿到這個

class OverallState(TypedDict):
    user_input: str              # 來自輸入
    graph_output: str            # 給輸出
    foo: str                     # 內部用，外部看不到
    bar: str                     # 內部用，外部看不到

builder = StateGraph(
    OverallState,
    input_schema=InputState,
    output_schema=OutputState,
)
```

**效果**：

- `graph.invoke({"user_input": "Hi"})` — 只需傳 `InputState` 的欄位
- 回傳值只包含 `OutputState` 的欄位：`{"graph_output": "..."}`
- 內部節點可以自由讀寫 `OverallState` 的所有欄位

### MessagesState：內建的訊息型 State

由於「對話訊息列表」是最常見的 State 結構，LangGraph 內建了 `MessagesState`：

```python
from langgraph.graph import MessagesState

# MessagesState 等同於：
# class MessagesState(TypedDict):
#     messages: Annotated[list[AnyMessage], add_messages]

# 通常會繼承它，加入自己的欄位
class MyState(MessagesState):
    documents: list[str]
    current_tool: str
```

`MessagesState` 的 `messages` 欄位使用 `add_messages` reducer，具有：
- **自動追加**新訊息
- **智慧更新**同 ID 的訊息（而非重複追加）
- **自動反序列化**（支援 dict 格式和 LangChain Message 物件）

```python
# 這兩種寫法都支援：
graph.invoke({"messages": [HumanMessage(content="你好")]})
graph.invoke({"messages": [{"type": "human", "content": "你好"}]})
```

## 4. Channel：State 中的個別欄位

### 定義

Channel 是 State Schema 中的**每一個獨立欄位**。每個 Channel 都有自己獨立的：

- **型別**（int、str、list 等）
- **Reducer 函數**（如何處理更新）
- **目前值**

### 概念圖

```
State（整體）
├── Channel: "messages"   → 型別: list[AnyMessage], Reducer: add_messages
├── Channel: "counter"    → 型別: int,              Reducer: 預設（覆蓋）
├── Channel: "tags"       → 型別: list[str],        Reducer: operator.add
└── Channel: "status"     → 型別: str,              Reducer: 預設（覆蓋）
```

### 為什麼要理解 Channel？

Channel 的概念來自 Google Pregel 的 message-passing 模型。在 Pregel 中，節點之間透過 **channel** 傳遞訊息。LangGraph 的實作是：

1. 每個 State 欄位就是一個 Channel
2. 節點的回傳值 → 寫入對應的 Channel
3. Channel 的 Reducer 決定新值如何與舊值合併
4. 合併後的 State → 傳給下一個節點

```
Node A 回傳 {"counter": 5, "tags": ["new"]}
              │                  │
              v                  v
     Channel "counter"    Channel "tags"
     Reducer: 覆蓋         Reducer: operator.add
     舊值: 3 → 新值: 5     舊值: ["old"] → 新值: ["old", "new"]
```

### 進階：Private Channel

節點可以宣告額外的 Channel，不在主 State Schema 中，用於節點之間的**內部通訊**：

```python
class OverallState(TypedDict):
    user_input: str
    result: str

class PrivateState(TypedDict):
    internal_data: str    # 這個 Channel 不在 OverallState 中

def node_a(state: OverallState) -> PrivateState:
    # 寫入 private channel
    return {"internal_data": "some intermediate value"}

def node_b(state: PrivateState) -> OverallState:
    # 讀取 private channel，寫入主 state
    return {"result": state["internal_data"]}
```

## 5. Reducer：定義 State 更新如何合併

### 定義

Reducer 是附加在每個 Channel 上的**合併策略函數**。當節點回傳更新值時，Reducer 決定這個新值如何與現有值結合。

### 預設 Reducer：直接覆蓋

如果不指定 Reducer，新值會**直接取代**舊值：

```python
class State(TypedDict):
    foo: int            # 無 Reducer → 覆蓋
    bar: list[str]      # 無 Reducer → 覆蓋
```

```
初始:  {"foo": 1, "bar": ["hi"]}
Node 回傳: {"foo": 2}
結果:  {"foo": 2, "bar": ["hi"]}      # foo 被覆蓋

Node 回傳: {"bar": ["bye"]}
結果:  {"foo": 2, "bar": ["bye"]}     # bar 整個被覆蓋（不是追加！）
```

### 使用 Annotated 指定 Reducer

用 `Annotated[型別, reducer_func]` 語法：

```python
from typing import Annotated
from operator import add

class State(TypedDict):
    foo: int                              # 無 Reducer → 覆蓋
    bar: Annotated[list[str], add]        # 用 operator.add → 串接
```

```
初始:  {"foo": 1, "bar": ["hi"]}
Node 回傳: {"bar": ["bye"]}
結果:  {"foo": 1, "bar": ["hi", "bye"]}  # bar 是串接，不是覆蓋！
```

### 常用 Reducer 一覽

| Reducer | 用途 | 範例 |
|---------|------|------|
| **（預設）** | 直接覆蓋舊值 | `foo: int` |
| **`operator.add`** | 串接 list 或加法 | `logs: Annotated[list[str], add]` |
| **`add_messages`** | 智慧管理訊息列表（追加 + ID 更新） | `messages: Annotated[list[AnyMessage], add_messages]` |
| **自定義函數** | 任意合併邏輯 | `Annotated[dict, my_merge_func]` |

### 自定義 Reducer 函數

Reducer 函數接收兩個參數：`(current_value, new_value)` → 回傳合併後的值。

```python
def keep_max(current: int, new: int) -> int:
    """只保留較大的值"""
    return max(current, new)

def merge_dicts(current: dict, new: dict) -> dict:
    """深度合併字典"""
    merged = {**current, **new}
    return merged

class State(TypedDict):
    score: Annotated[int, keep_max]        # 只保留最高分
    metadata: Annotated[dict, merge_dicts]  # 合併字典
```

### add_messages Reducer 的智慧行為

`add_messages` 是 LangGraph 內建的 Reducer，專為對話訊息設計：

```python
from langgraph.graph.message import add_messages

class State(TypedDict):
    messages: Annotated[list[AnyMessage], add_messages]
```

**行為一：新訊息 → 追加**

```python
# 初始: messages = [HumanMessage("Hi")]
# Node 回傳: {"messages": [AIMessage("Hello!")]}
# 結果: messages = [HumanMessage("Hi"), AIMessage("Hello!")]
```

**行為二：同 ID 訊息 → 更新（覆蓋）**

```python
# 初始: messages = [AIMessage(id="msg-1", content="Draft")]
# Node 回傳: {"messages": [AIMessage(id="msg-1", content="Final")]}
# 結果: messages = [AIMessage(id="msg-1", content="Final")]
# → 同 ID 的訊息被更新，而非重複追加
```

這在 Human-in-the-Loop 場景中特別重要——人類修改某則訊息時，用同 ID 更新而非追加。

### Overwrite：繞過 Reducer

有時你已經設定了 Reducer，但在某個節點想直接覆蓋而非合併：

```python
from langgraph.graph.state import Overwrite

def reset_node(state: State) -> dict:
    # 即使 messages 有 add_messages reducer，
    # Overwrite 也會直接取代整個列表
    return {"messages": Overwrite([SystemMessage("Reset")])}
```

## 6. Node：工作單元

### 定義

Node 是 LangGraph 中的**工作單元**——一個 Python 函數，負責讀取 State、執行邏輯、回傳更新。

> Nodes and Edges are nothing more than functions — they can contain an LLM or just good ol' code.
> — LangGraph 官方文件

### Node 函數的簽名

Node 函數可以接收一到三個參數：

```python
# 最簡單：只接收 state
def plain_node(state: MyState) -> dict:
    return {"counter": state["counter"] + 1}

# 加上 config：取得 thread_id、tags 等設定
from langchain_core.runnables import RunnableConfig

def node_with_config(state: MyState, config: RunnableConfig) -> dict:
    thread_id = config["configurable"]["thread_id"]
    return {"result": f"Thread: {thread_id}"}

# 加上 runtime：取得 store、stream_writer、context 等
from langgraph.runtime import Runtime

def node_with_runtime(state: MyState, runtime: Runtime) -> dict:
    # runtime.context → 執行時注入的依賴
    # runtime.store → Memory Store
    return {"result": "done"}
```

### 註冊 Node

```python
builder = StateGraph(MyState)

# 方式一：指定名稱
builder.add_node("my_step", my_function)

# 方式二：自動用函數名作為節點名
builder.add_node(my_function)
# 等同於 builder.add_node("my_function", my_function)
```

### Node 的回傳值

Node 函數回傳一個 `dict`，代表對 State 的**更新**（不是完整 State）：

```python
class State(TypedDict):
    a: int
    b: str
    c: list[str]

def my_node(state: State) -> dict:
    # 只更新 a，b 和 c 不變
    return {"a": state["a"] + 1}
```

Node 也可以回傳 `Command` 物件（見第 9 節），或在 Map-Reduce 場景中回傳 `list[Send]`。

### Node 可以做什麼？

Node 就是普通的 Python 函數，可以做**任何事**：

```
┌─────────────────────────────────────────────────────┐
│                 Node 可執行的操作                      │
├─────────────────────────────────────────────────────┤
│  呼叫 LLM       │  model.invoke(messages)           │
│  呼叫工具        │  search(query), calculator(expr)  │
│  資料處理        │  清洗、轉換、格式化                  │
│  API 呼叫       │  requests.get(url)                │
│  資料庫操作      │  db.query(sql)                    │
│  條件邏輯        │  if/else 判斷                     │
│  人機互動        │  interrupt() 暫停等待              │
│  什麼都不做      │  直接回傳 state（pass-through）    │
└─────────────────────────────────────────────────────┘
```

### Node Caching

LangGraph 支援根據輸入快取節點結果，避免重複計算：

```python
from langgraph.types import CachePolicy
from langgraph.cache.memory import InMemoryCache

builder.add_node(
    "expensive_node",
    expensive_function,
    cache_policy=CachePolicy(ttl=60),  # 快取 60 秒
)

graph = builder.compile(cache=InMemoryCache())
```

## 7. Edge：節點之間的連接

### 定義

Edge 決定了「一個節點執行完之後，下一個要執行哪個節點」。Edge 有四種類型：

### 7.1 Normal Edge（固定邊）

永遠從 A 到 B，無條件：

```python
builder.add_edge("node_a", "node_b")
```

```
┌────────┐         ┌────────┐
│ node_a │ ──────> │ node_b │
└────────┘         └────────┘
  永遠走這條路
```

### 7.2 Conditional Edge（條件邊）

根據**路由函數**的回傳值決定下一步：

```python
def route_function(state: MyState) -> str:
    if state["score"] > 80:
        return "pass_node"
    else:
        return "retry_node"

builder.add_conditional_edges(
    "evaluate",          # 從哪個節點出發
    route_function,      # 路由函數
    ["pass_node", "retry_node"],  # 可能的目標（用於圖視覺化）
)
```

```
                    ┌────────────┐
              ┌────>│  pass_node │    (score > 80)
┌──────────┐  │     └────────────┘
│ evaluate │──┤
└──────────┘  │     ┌────────────┐
              └────>│ retry_node │    (score <= 80)
                    └────────────┘
```

也可以用 dict 映射路由函數的回傳值：

```python
builder.add_conditional_edges(
    "evaluate",
    route_function,
    {True: "pass_node", False: "retry_node"},  # 回傳值 → 節點名
)
```

**重要**：路由函數也是普通的 Python 函數，接收 `state` 作為參數。它可以包含任何邏輯——包括呼叫 LLM 來決定路由。

### 7.3 Entry Point（入口邊）

從虛擬的 `START` 節點出發，決定 Graph 的起點：

```python
from langgraph.graph import START

# 固定入口
builder.add_edge(START, "first_node")

# 條件入口——根據輸入決定從哪裡開始
builder.add_conditional_edges(START, route_function)
```

### 7.4 平行邊

一個節點可以有**多條出邊**，所有目標節點會在同一個 super-step 中**平行執行**：

```python
builder.add_edge("start_node", "branch_a")
builder.add_edge("start_node", "branch_b")
builder.add_edge("start_node", "branch_c")
# branch_a, branch_b, branch_c 同時執行（平行）
```

```
                ┌───────────┐
           ┌───>│ branch_a  │
           │    └───────────┘
┌────────┐ │    ┌───────────┐
│ start  │─┼───>│ branch_b  │   ← 同一 super-step，平行執行
└────────┘ │    └───────────┘
           │    ┌───────────┐
           └───>│ branch_c  │
                └───────────┘
```

### 7.5 Send（動態邊）

用於 **Map-Reduce** 模式——在執行時動態建立分支，數量不固定：

```python
from langgraph.types import Send

def fan_out(state: OverallState):
    """根據 state 中的項目數量，動態產生平行分支"""
    return [
        Send("process_item", {"item": item})
        for item in state["items"]
    ]

builder.add_conditional_edges("collector", fan_out, ["process_item"])
```

與一般 Edge 的差異：

| 特性 | Normal / Conditional Edge | Send |
|------|--------------------------|------|
| 分支數量 | 編譯時固定 | 執行時動態決定 |
| 目標 State | 共享同一個 State | 每個分支可以有獨立 State |
| 用途 | 一般控制流 | Map-Reduce、批次處理 |

## 8. START 與 END 節點：標記執行起點與終止

### START 節點

`START` 是一個**虛擬節點**，代表「使用者輸入進入圖的入口」。它不執行任何邏輯，只作為起點標記：

```python
from langgraph.graph import START

builder.add_edge(START, "first_node")   # 圖從 first_node 開始
```

### END 節點

`END` 是一個**虛擬節點**，代表「圖的執行終止點」。當執行流到達 END，圖停止並回傳最終 State：

```python
from langgraph.graph import END

builder.add_edge("last_node", END)   # last_node 完成後，圖結束
```

### 在條件邊中使用 END

END 常用在條件路由中，作為「終止」的選項：

```python
def should_continue(state: MyState) -> str:
    if state["done"]:
        return END           # 結束圖的執行
    return "next_node"       # 繼續執行

builder.add_conditional_edges("check", should_continue, ["next_node", END])
```

### 多個 END 出口

一個圖可以有**多個通往 END 的路徑**——任何一條到達 END 的路徑都會使圖終止：

```
START ─> node_a ─> check ──┬──> node_b ──> END   （路徑 1）
                           │
                           └──> END               （路徑 2：直接結束）
```

### 執行終止的完整條件

根據 Pregel 模型，圖的執行會在以下條件終止：

1. **到達 END 節點**：顯式終止
2. **所有節點 inactive**：沒有任何節點有待處理的訊息
3. **達到 recursion_limit**：超過最大 super-step 次數（預設 1000），拋出 `GraphRecursionError`

## 9. Command：結合控制流與 State 更新

### 定義

`Command` 是 LangGraph 的進階原語，讓你在**一次操作中同時做兩件事**：

1. **更新 State**（`update`）
2. **決定下一步去哪**（`goto`）

傳統做法需要分開處理——節點回傳 State 更新，然後用條件邊決定路由。`Command` 把這兩步合為一。

### 基本用法

```python
from langgraph.types import Command
from typing import Literal

def my_node(state: MyState) -> Command[Literal["node_b", "node_c"]]:
    if state["score"] > 80:
        return Command(
            update={"status": "passed"},    # 更新 State
            goto="node_b",                  # 導向 node_b
        )
    else:
        return Command(
            update={"status": "failed"},    # 更新 State
            goto="node_c",                  # 導向 node_c
        )
```

### Command 的四個參數

| 參數 | 用途 | 使用場景 |
|------|------|---------|
| `update` | 更新 State（等同節點回傳 dict） | Node 回傳 |
| `goto` | 導向指定節點（等同條件邊） | Node 回傳 |
| `graph` | 指定目標圖（`Command.PARENT` 導向父圖） | 子圖中的 Node 回傳 |
| `resume` | 提供值以恢復被 `interrupt()` 暫停的執行 | `invoke()` / `stream()` 的輸入 |

### Command vs 條件邊

```python
# ===== 傳統做法：節點 + 條件邊 =====

def my_node(state: MyState) -> dict:
    if state["score"] > 80:
        return {"status": "passed", "route": "node_b"}
    return {"status": "failed", "route": "node_c"}

def route(state: MyState) -> str:
    return state["route"]

builder.add_conditional_edges("my_node", route, ["node_b", "node_c"])

# ===== Command 做法：合二為一 =====

def my_node(state: MyState) -> Command[Literal["node_b", "node_c"]]:
    if state["score"] > 80:
        return Command(update={"status": "passed"}, goto="node_b")
    return Command(update={"status": "failed"}, goto="node_c")

# 不需要 add_conditional_edges！
```

### Command 的型別標註

**重要**：使用 Command 時必須標註回傳型別，列出所有可能的目標節點：

```python
# 必須標註 Command[Literal["可能的目標節點"]]
def my_node(state: State) -> Command[Literal["node_a", "node_b"]]:
    ...
```

這個型別標註的用途：
1. 讓 LangGraph 知道這個節點可能導向哪些節點（用於**圖視覺化**）
2. 提供 IDE 的型別檢查支援

### Command 與靜態邊的共存

**注意**：Command 只增加**動態邊**，不會取消已定義的靜態邊：

```python
builder.add_edge("node_a", "node_b")  # 靜態邊

def node_a(state: State) -> Command[Literal["node_c"]]:
    return Command(goto="node_c")      # 動態邊

# 結果：node_b 和 node_c 都會執行！
```

### Command 用於 interrupt 恢復

```python
from langgraph.types import Command, interrupt

# 在節點中暫停
def review_node(state: State):
    answer = interrupt("請確認是否繼續？")  # 暫停，等待人類回應
    return {"decision": answer}

# 恢復執行
result = graph.invoke(Command(resume="yes"), config)
# "yes" 會成為 interrupt() 的回傳值
```

### Command 用於子圖導航

```python
def subgraph_node(state: State) -> Command[Literal["parent_node"]]:
    return Command(
        update={"result": "done"},
        goto="parent_node",         # 父圖中的節點
        graph=Command.PARENT,       # 指定導向父圖
    )
```

## 10. 原語之間的關係圖

```
┌──────────────────────────────────────────────────────────────┐
│                        StateGraph                            │
│                     （圖的容器/建構器）                         │
│                                                              │
│   定義 ──> State Schema                                      │
│              │                                               │
│              ├── Channel: "messages"  ← Reducer: add_messages│
│              ├── Channel: "counter"   ← Reducer: 覆蓋        │
│              └── Channel: "tags"      ← Reducer: operator.add│
│                                                              │
│   註冊 ──> Nodes                                             │
│              ├── "llm_call"    (Python 函數)                  │
│              ├── "tool_node"   (Python 函數)                  │
│              └── "summarize"   (Python 函數)                  │
│                                                              │
│   宣告 ──> Edges                                             │
│              ├── START ──────────> "llm_call"  (固定邊)       │
│              ├── "llm_call" ─?──> "tool_node"  (條件邊)      │
│              ├── "llm_call" ─?──> END          (條件邊)      │
│              ├── "tool_node" ───> "llm_call"   (固定邊)      │
│              └── Command(goto=...) ──> 動態邊                │
│                                                              │
│   compile() ──> 可執行的 Graph                                │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 執行時的資料流

```
invoke({"messages": [HumanMessage("Hi")]})
    │
    v
 [START] ──────> [llm_call] ──────> [tool_node] ──────> [llm_call] ──> [END]
                     │                   │                   │
                     │                   │                   │
              讀取 State            讀取 State          讀取 State
              執行邏輯              執行邏輯            執行邏輯
              回傳更新              回傳更新            回傳更新
                     │                   │                   │
                     v                   v                   v
              Reducer 合併          Reducer 合併        Reducer 合併
              更新 State            更新 State          更新 State
                     │                   │                   │
                  checkpoint          checkpoint         checkpoint
               (Super-step 0)     (Super-step 1)     (Super-step 2)
```

## 重點摘要

| 原語 | 一句話說明 | 關鍵 API |
|------|-----------|---------|
| **StateGraph** | 圖的容器，定義 Schema、承載 Node 和 Edge | `StateGraph(Schema)`, `.compile()` |
| **State** | 共享資料結構，節點間的「記憶」 | `TypedDict`, `dataclass`, `BaseModel` |
| **Channel** | State 中的每個欄位，各自獨立管理 | State Schema 中的每個 key |
| **Reducer** | 定義 Channel 收到更新時如何合併 | `Annotated[type, reducer]` |
| **Node** | 工作單元，一個 Python 函數 | `add_node("name", func)` |
| **Edge** | 節點間的連接，決定執行順序 | `add_edge()`, `add_conditional_edges()` |
| **START / END** | 虛擬節點，標記圖的入口與出口 | `from langgraph.graph import START, END` |
| **Command** | 在一次操作中同時更新 State 與控制流 | `Command(update=, goto=, resume=)` |

### 選擇指南

```
需要固定的執行順序？            → Normal Edge
需要根據 State 決定下一步？      → Conditional Edge
需要同時更新 State 又決定路由？   → Command
需要動態數量的平行分支？         → Send
需要跨子圖導航？                → Command(graph=Command.PARENT)
需要暫停等待人類？              → interrupt() + Command(resume=)
```
