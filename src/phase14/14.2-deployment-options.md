## 1. 部署選項總覽

```
┌────────────────────────────────────────────────────────┐
│               LangGraph 部署選項                        │
├──────────────────┬─────────────────────────────────────┤
│                  │                                      │
│  本地開發         │  langgraph dev  (無 Docker)          │
│                  │  langgraph up   (Docker)             │
│                  │                                      │
├──────────────────┼─────────────────────────────────────┤
│                  │                                      │
│  Cloud 部署      │  LangSmith Cloud (全託管)            │
│                  │  從 GitHub repo 自動部署              │
│                  │                                      │
├──────────────────┼─────────────────────────────────────┤
│                  │                                      │
│  Self-hosted     │  Control Plane (混合式)              │

│                  │                                      │
├──────────────────┼─────────────────────────────────────┤
│                  │                                      │
```

## 2. 本地伺服器（CLI + Studio）

### 方式一：langgraph dev（輕量開發）

不需要 Docker，適合快速迭代：

```bash
# 安裝 CLI
pip install langgraph-cli

# 啟動本地開發 server
langgraph dev

# 輸出：
# Starting LangGraph API server...
# Ready at http://localhost:2024
# Studio: https://smith.langchain.com/studio/?baseUrl=http://localhost:2024
```

### 方式二：langgraph up（Docker 模式）

更接近生產環境，需要 Docker：

```bash
# 確認 Docker 運行中
docker --version

# 啟動（含 PostgreSQL + Redis）
langgraph up

# 健康檢查
curl http://localhost:8123/ok
# {"ok": true}
```

### 連接 Studio

Studio 可以連接到任何正在運行的本地 Agent Server：

```
┌─────────────┐         ┌──────────────────┐
│  LangGraph  │  HTTP   │   Agent Server   │
│   Studio    │ ──────> │ localhost:2024    │
│  (瀏覽器)   │         │                  │
└─────────────┘         └──────────────────┘
```

## 3. Cloud 部署（LangSmith Cloud）

### 特點

- **全託管**：LangSmith 處理所有基礎設施、擴展、營運
- **從 GitHub 部署**：連結 repo，自動 build 和部署
- **專為 Agent 設計**：內建 stateful、長時間執行支援

### 部署步驟

```
步驟 1: 準備 GitHub Repository
├── 確保有 langgraph.json
├── 確保有 agent 程式碼
└── push 到 GitHub

步驟 2: LangSmith UI 部署
├── 登入 LangSmith
├── 左側選單 → Deployments
├── + New Deployment
├── 連接 GitHub 帳號
├── 選擇 Repository
└── Submit（約 15 分鐘完成）

步驟 3: 取得 API URL
└── 在 Deployment 詳情頁複製 API URL
```

### 使用 SDK 連接 Cloud 部署

```python
"""
連接 LangSmith Cloud 部署的 Agent
"""
from langgraph_sdk import get_sync_client

# === 建立連接 ===
client = get_sync_client(
    url="https://your-deployment-url.langsmith.com",
    api_key="your-langsmith-api-key"
)

# === 執行 Agent（Threadless run = 無狀態） ===
for chunk in client.runs.stream(
    None,        # Threadless run
    "agent",     # langgraph.json 中定義的 graph 名稱
    input={
        "messages": [
            {"role": "human", "content": "What is LangGraph?"}
        ]
    },
    stream_mode="updates",
):
    print(f"Event: {chunk.event}")
    print(f"Data: {chunk.data}\n")


# === 有狀態執行（使用 Thread） ===
thread = client.threads.create()

result = client.runs.wait(
    thread_id=thread["thread_id"],
    assistant_id="agent",
    input={
        "messages": [
            {"role": "human", "content": "記住我叫小明"}
        ]
    },
)
print(f"結果: {result}")

# 同一 thread 繼續對話
result2 = client.runs.wait(
    thread_id=thread["thread_id"],
    assistant_id="agent",
    input={
        "messages": [
            {"role": "human", "content": "我剛才說我叫什麼？"}
        ]
    },
)
print(f"結果: {result2}")
```

## 4. Self-hosted（自託管）

### 方式一：Control Plane（混合式）

保留 LangSmith UI 管理能力，但 Agent Server 運行在你的基礎設施上。

```
┌──────────────────────┐      ┌───────────────────────┐
│   LangSmith Cloud    │      │   你的基礎設施          │
│  (Control Plane)     │ <──> │  (Data Plane)          │
│  - UI 管理           │      │  - Agent Server        │
│  - 部署管理          │      │  - PostgreSQL          │
│  - Monitoring        │      │  - Redis               │
└──────────────────────┘      └───────────────────────┘
```

### 方式二：Standalone Server（完全獨立）

無需 LangSmith UI 或 Control Plane，最輕量的自託管選項。

**必要環境變數：**

| 變數 | 說明 |
|------|------|
| `REDIS_URI` | Redis 連線 URI（用於 streaming pub-sub） |
| `DATABASE_URI` | PostgreSQL 連線 URI（用於 state 持久化） |
| `LANGSMITH_API_KEY` | LangSmith API Key |
| `LANGGRAPH_CLOUD_LICENSE_KEY` | 授權金鑰（啟動時驗證一次） |

## 5. Docker 容器部署

### 步驟一：建構 Docker Image

```bash
# 用 CLI 建構
langgraph build -t my-agent:latest

# 或產生 Dockerfile 後自行建構
langgraph dockerfile > Dockerfile
docker build -t my-agent:latest .
```

### 步驟二：Docker 單獨啟動

```bash
docker run \
    --env-file .env \
    -p 8123:8000 \
    -e REDIS_URI="redis://redis:6379" \
    -e DATABASE_URI="postgres://user:pass@postgres:5432/db" \
    -e LANGSMITH_API_KEY="lsv2_..." \
    my-agent:latest
```

### 步驟三：Docker Compose（推薦）

```yaml
# docker-compose.yml
# 完整的 Agent Server + PostgreSQL + Redis

volumes:
    langgraph-data:
        driver: local

services:
    langgraph-redis:
        image: redis:6
        healthcheck:
            test: redis-cli ping
            interval: 5s
            timeout: 1s
            retries: 5

    langgraph-postgres:
        image: postgres:16
        ports:
            - "5432:5432"
        environment:
            POSTGRES_DB: postgres
            POSTGRES_USER: postgres
            POSTGRES_PASSWORD: postgres
        volumes:
            - langgraph-data:/var/lib/postgresql/data
        healthcheck:
            test: pg_isready -U postgres
            start_period: 10s
            timeout: 1s
            retries: 5
            interval: 5s

    langgraph-api:
        image: ${IMAGE_NAME}
        ports:
            - "8123:8000"
        depends_on:
            langgraph-redis:
                condition: service_healthy
            langgraph-postgres:
                condition: service_healthy
        env_file:
            - .env
        environment:
            REDIS_URI: redis://langgraph-redis:6379
            LANGSMITH_API_KEY: ${LANGSMITH_API_KEY}
            DATABASE_URI: >-
                postgres://postgres:postgres@langgraph-postgres:5432/postgres?sslmode=disable
```

### Kubernetes 部署

使用官方 Helm Chart：

```bash
# 安裝 Helm Chart（v0.2.6+）
helm install my-agent langgraph/agent-server \
    --set image.repository=my-agent \
    --set image.tag=latest \
    --set redis.uri="redis://redis:6379" \
    --set database.uri="postgres://..." \
    --set langsmith.apiKey="lsv2_..."
```

## 6. RemoteGraph：連接已部署的 Graph

### 什麼是 RemoteGraph

RemoteGraph 是一個客戶端介面，讓你用與 `CompiledGraph` **完全相同的 API** 與遠端部署互動。

```
┌─────────────────┐              ┌──────────────────────┐
│  你的應用程式     │   HTTP API   │   Agent Server       │
│                  │              │                      │
│  RemoteGraph     │ ──────────> │  已部署的 Graph       │
│  .invoke()       │              │                      │
│  .stream()       │ <────────── │  執行結果             │
│  .get_state()    │              │                      │
└─────────────────┘              └──────────────────────┘
```

### 適用場景

- **開發 → 生產無縫切換**：本地用 `CompiledGraph`，部署後用 `RemoteGraph`
- **Thread 持久化**：跨呼叫保持對話狀態
- **子圖嵌入**：將已部署的 graph 作為另一個 graph 的子圖
- **可重用工作流**：將已部署的 graph 作為節點或工具

### 完整範例

```python
"""
RemoteGraph 完整使用範例
展示初始化、invoke、stream、state 管理、子圖嵌入
"""
from langgraph.pregel.remote import RemoteGraph
from langgraph_sdk import get_sync_client
from langgraph.graph import StateGraph, MessagesState, START


# ============================================================
# 範例 1：基本使用 — invoke 和 stream
# ============================================================
def basic_usage():
    """透過 URL 初始化 RemoteGraph"""
    remote_graph = RemoteGraph(
        "agent",                           # graph 名稱或 assistant ID
        url="http://localhost:2024",       # 部署 URL
        # api_key="your-key"              # Cloud 部署需要
    )

    # invoke — 同步呼叫，等待完整結果
    result = remote_graph.invoke({
        "messages": [{"role": "user", "content": "你好！"}]
    })
    print(f"Invoke 結果: {result}")

    # stream — 串流取得結果
    for chunk in remote_graph.stream({
        "messages": [{"role": "user", "content": "天氣如何？"}]
    }):
        print(f"Stream chunk: {chunk}")


# ============================================================
# 範例 2：Thread 持久化
# ============================================================
def thread_persistence():
    """使用 thread 保持對話狀態"""
    url = "http://localhost:2024"
    sync_client = get_sync_client(url=url)
    remote_graph = RemoteGraph("agent", url=url)

    # 建立 thread
    thread = sync_client.threads.create()
    config = {"configurable": {"thread_id": thread["thread_id"]}}

    # 第一次對話
    result1 = remote_graph.invoke(
        {"messages": [{"role": "user", "content": "我叫小明"}]},
        config=config,
    )

    # 第二次對話（同一 thread，保持記憶）
    result2 = remote_graph.invoke(
        {"messages": [{"role": "user", "content": "我叫什麼名字？"}]},
        config=config,
    )

    # 取得 thread state
    state = remote_graph.get_state(config)
    print(f"Thread state: {state}")


# ============================================================
# 範例 3：作為子圖嵌入
# ============================================================
def subgraph_embedding():
    """將 RemoteGraph 作為子圖嵌入到父圖中"""
    remote_graph = RemoteGraph("agent", url="http://localhost:2024")

    # 建立父圖
    builder = StateGraph(MessagesState)
    # 直接將 RemoteGraph 加為節點
    builder.add_node("remote_agent", remote_graph)
    builder.add_edge(START, "remote_agent")

    parent_graph = builder.compile()

    # 執行父圖（會自動呼叫遠端 agent）
    result = parent_graph.invoke({
        "messages": [{"role": "user", "content": "幫我查天氣"}]
    })
    print(f"父圖結果: {result}")

    # 串流（含子圖輸出）
    for chunk in parent_graph.stream(
        {"messages": [{"role": "user", "content": "你好"}]},
        subgraphs=True,
    ):
        print(f"Chunk: {chunk}")


if __name__ == "__main__":
    basic_usage()
```

> 📄 完整範例程式碼：[14.2-example-remote-graph.py](./14.2-example-remote-graph.py)

> **注意**：RemoteGraph 設計為呼叫**其他**部署的 graph。不要用 RemoteGraph 呼叫自身所在的部署，否則可能導致死鎖和資源耗盡。

## 7. 部署選項比較

| 面向 | langgraph dev | Cloud | Self-hosted (Control Plane) | Standalone |
|------|---------------|-------|-----------------------------|-----------|
| **基礎設施管理** | 無需 | LangSmith 管理 | 混合 | 完全自管 |
| **需要 Docker** | 否 | 否 | 是 | 是 |
| **UI 管理** | Studio | LangSmith UI | LangSmith UI | 無 |
| **Auto Scaling** | 否 | 是 | 依基礎設施 | 依基礎設施 |
| **持久化** | 記憶體 | PostgreSQL | PostgreSQL | PostgreSQL |
| **適合** | 開發 | 生產（快速上線） | 生產（合規需求） | 生產（最大控制） |
| **費用** | 免費 | 按用量 | 授權費 | 授權費 |

---

## 重點摘要

| 概念 | 重點 |
|------|------|
| **langgraph dev** | 最快速的本地開發方式，不需 Docker |
| **Cloud 部署** | 從 GitHub 一鍵部署，LangSmith 全託管 |
| **Self-hosted** | Control Plane（混合式）或 Standalone（獨立式） |
| **Docker Compose** | 推薦的自託管方式，包含 Agent Server + PostgreSQL + Redis |
| **Kubernetes** | 使用官方 Helm Chart 部署到 K8s 叢集 |
| **RemoteGraph** | 用 CompiledGraph 相同 API 存取遠端部署 |
| **子圖嵌入** | RemoteGraph 可直接作為另一個 graph 的節點 |
