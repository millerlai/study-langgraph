## 1. Platform 概觀

### 什麼是 LangGraph Platform

LangGraph Platform 是一套完整的**部署與管理解決方案**，專為 LangGraph 建構的 stateful agent 應用設計。它提供從本地開發、測試、到正式環境部署所需的所有工具。

```
┌─────────────────────────────────────────────────────────────┐
│                   LangGraph Platform                         │
│                                                              │
│  ┌──────────┐   ┌──────────────────────┐   ┌────────────┐  │
│  │ LangGraph│   │    Agent Server       │   │  LangGraph │  │
│  │   CLI    │──>│  (API + Runtime)      │<──│   Studio   │  │
│  └──────────┘   │  - Execution          │   └────────────┘  │
│                 │  - State Management   │                    │
│                 │  - Persistence        │   ┌────────────┐  │
│                 │  - Background Runs    │<──│  SDK        │  │
│                 └──────────────────────┘   │  (Python/JS)│  │
│                         ^                   └────────────┘  │
│                         |                                    │
│                 ┌───────┴───────┐   ┌─────────────────┐    │
│                 │  RemoteGraph  │   │  Control Plane   │    │
│                 │  (客戶端整合) │   │  (UI + 管理API)  │    │
│                 └───────────────┘   └─────────────────┘    │
└─────────────────────────────────────────────────────────────┘
```

### 核心價值

| 面向 | 說明 |
|------|------|
| **開發效率** | CLI + Studio 提供快速的本地開發與視覺化除錯 |
| **生產就緒** | Agent Server 內建 persistence、background run、streaming |
| **彈性部署** | 支援 Cloud、Self-hosted、Standalone 多種部署模式 |
| **統一介面** | SDK + RemoteGraph 讓本地開發與遠端部署使用相同 API |

### 元件關係圖

```
                    ┌──────────────────┐

                             │ creates

                    └──────────────────┘
```

### 元件一覽表

| 元件 | 角色 | 關鍵功能 |
|------|------|----------|
| **Agent Server** | API Runtime | 執行 graph、管理 state、persistence |
| **LangGraph CLI** | 命令列工具 | 建構、打包、本地啟動 server |
| **Studio** | 視覺化 IDE | 除錯、互動、state 觀察 |
| **Python/JS SDK** | 程式化客戶端 | 從應用程式呼叫已部署的 graph |
| **RemoteGraph** | 無縫橋梁 | 用 CompiledGraph 相同 API 存取遠端 graph |
| **Control Plane** | 管理介面 | 建立、更新、管理部署 |
| **Data Plane** | 執行層 | Agent Server + 支援服務 (PostgreSQL, Redis) |

## 3. Agent Server

### 功能定義

Agent Server 定義了一套**標準化的 API 與 Runtime**，用來部署和執行 graph/agent。它處理：

- **執行管理**：graph invocation、streaming、background runs
- **狀態管理**：thread 和 checkpoint 的持久化
- **持久化儲存**：自動管理 PostgreSQL checkpointer
- **背景任務**：長時間運行的 graph 不阻塞 API 回應

### API 端點概覽

```
Agent Server API
├── /threads          # Thread 管理
│   ├── POST /        # 建立 thread
│   ├── GET /{id}     # 取得 thread
│   └── GET /{id}/state  # 取得 thread state
├── /runs             # 執行管理
│   ├── POST /stream  # Streaming 執行
│   ├── POST /wait    # 同步等待結果
│   └── POST /        # Background 執行
├── /assistants       # Assistant 管理
│   ├── POST /search  # 搜尋 assistants
│   └── GET /{id}     # 取得 assistant 資訊
└── /crons            # 排程任務
    ├── POST /        # 建立 cron job
    └── DELETE /{id}  # 刪除 cron job
```

## 4. LangGraph CLI

### 安裝

```bash
pip install langgraph-cli
```

### 核心命令

| 命令 | 說明 |
|------|------|
| `langgraph dev` | 啟動輕量級本地開發 server（不需 Docker） |
| `langgraph build` | 建構 Docker image |
| `langgraph up` | 在 Docker 中啟動 Agent Server |
| `langgraph deploy` | 建構並部署到 LangSmith Deployments |
| `langgraph dockerfile` | 產生 Dockerfile 供自訂建構 |

### 完整範例：本地開發啟動

```python
# === 步驟 1：安裝 CLI ===
# pip install langgraph-cli

# === 步驟 2：確認 langgraph.json 存在 ===
# 在專案根目錄建立 langgraph.json（見「應用程式結構」章節）

# === 步驟 3：啟動本地開發 server ===
# langgraph dev
# 輸出：
#   Starting LangGraph API server...
#   Ready at http://localhost:2024
#   Studio available at https://smith.langchain.com/studio/?baseUrl=http://localhost:2024

# === 步驟 4：測試 API ===
# curl --request GET --url http://localhost:2024/ok
# 輸出：{"ok": true}
```

## 5. LangGraph Studio

Studio 是一個專門用於 agent 開發的**視覺化 IDE**，提供：

- **Graph 架構視覺化**：看見節點與邊的結構
- **互動式執行**：直接在 UI 中執行 agent
- **State 觀察**：即時查看每個節點的輸入/輸出
- **時間旅行除錯**：回溯檢視過去的 checkpoint
- **Thread 管理**：建立、切換、管理對話 thread

### 兩種模式

| 模式 | 適用場景 | 功能 |
|------|----------|------|
| **Graph Mode** | 開發除錯 | 完整功能：節點追蹤、中間 state、LangSmith 整合 |
| **Chat Mode** | 測試互動 | 簡化 UI，適合測試 chat-based agent |

## 6. Python/JS SDK

### 安裝

```bash
pip install langgraph-sdk
```

### 基本使用

```python
"""
LangGraph SDK 基本使用範例
展示如何透過 SDK 與已部署的 Agent Server 互動
"""
from langgraph_sdk import get_client, get_sync_client

# === 非同步客戶端 ===
async def async_example():
    client = get_client(
        url="http://localhost:2024",
        api_key="your-langsmith-api-key"  # 部署環境需要
    )

    # 建立 thread
    thread = await client.threads.create()
    print(f"Thread ID: {thread['thread_id']}")

    # 執行 graph（streaming）
    async for chunk in client.runs.stream(
        thread_id=thread["thread_id"],
        assistant_id="agent",  # langgraph.json 中定義的名稱
        input={
            "messages": [{"role": "human", "content": "你好！"}]
        },
        stream_mode="updates",
    ):
        print(f"Event: {chunk.event}")
        print(f"Data: {chunk.data}\n")


# === 同步客戶端 ===
def sync_example():
    client = get_sync_client(url="http://localhost:2024")

    # Threadless run（無狀態執行）
    for chunk in client.runs.stream(
        None,       # thread_id=None 表示無狀態執行
        "agent",
        input={
            "messages": [{"role": "human", "content": "Hello!"}]
        },
        stream_mode="updates",
    ):
        print(f"Event: {chunk.event}, Data: {chunk.data}")


if __name__ == "__main__":
    sync_example()
```

## 7. 應用程式結構

### 必要檔案

一個 LangGraph 應用程式需要以下結構：

```
my-agent/
├── langgraph.json      # 主設定檔（必要）
├── agent.py            # Graph 定義
├── requirements.txt    # 或 pyproject.toml
├── .env                # 環境變數（API keys 等）
└── auth.py             # 自訂認證（選用）
```

### langgraph.json 設定檔

```json
{
    "dependencies": ["."],
    "graphs": {
        "agent": "./agent.py:graph"
    },
    "env": ".env",
    "auth": {
        "path": "./auth.py:auth"
    },
    "store": {
        "index": {
            "embed": "openai:text-embeddings-3-small",
            "dims": 1536,
            "fields": ["$"]
        }
    }
}
```

### 設定欄位說明

| 欄位 | 必要 | 說明 |
|------|------|------|
| `dependencies` | 是 | Python 套件依賴，`"."` 表示本地套件 |
| `graphs` | 是 | Graph 名稱對應到模組路徑（`檔案:變數名`） |
| `env` | 否 | `.env` 檔案路徑 |
| `auth` | 否 | 自訂認證模組路徑 |
| `store` | 否 | Memory Store 設定（含語義搜尋） |
| `http` | 否 | HTTP 設定（如 `disable_webhooks`） |

## 8. 完整範例：從建立到啟動

### agent.py

```python
"""
完整的 LangGraph Agent 應用程式
可直接用 langgraph dev 或 langgraph up 啟動
"""
from typing import Annotated, Literal
from typing_extensions import TypedDict
from operator import add

from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages
from langchain_core.messages import AnyMessage


# === 1. 定義 State ===
class AgentState(TypedDict):
    """Agent 的共享狀態"""
    messages: Annotated[list[AnyMessage], add_messages]
    step_count: int


# === 2. 定義 Node 函數 ===
def greet(state: AgentState) -> dict:
    """歡迎節點：產生歡迎訊息"""
    return {
        "messages": [{"role": "assistant", "content": "你好！我是你的 AI 助手。"}],
        "step_count": state.get("step_count", 0) + 1,
    }


def process(state: AgentState) -> dict:
    """處理節點：回覆使用者訊息"""
    last_msg = state["messages"][-1]
    content = getattr(last_msg, "content", str(last_msg))
    return {
        "messages": [
            {"role": "assistant", "content": f"收到您的訊息：{content}，正在處理中..."}
        ],
        "step_count": state.get("step_count", 0) + 1,
    }


def route(state: AgentState) -> Literal["process", "__end__"]:
    """路由邏輯：決定下一步"""
    if state.get("step_count", 0) >= 3:
        return "__end__"
    return "process"


# === 3. 建構 Graph ===
builder = StateGraph(AgentState)

builder.add_node("greet", greet)
builder.add_node("process", process)

builder.add_edge(START, "greet")
builder.add_conditional_edges("greet", route)
builder.add_edge("process", END)

# === 4. 編譯（供 Agent Server 使用，不需指定 checkpointer）===
graph = builder.compile()
```

### langgraph.json

```json
{
    "dependencies": ["."],
    "graphs": {
        "agent": "./agent.py:graph"
    },
    "env": ".env"
}
```

### 啟動與測試

```bash
# 本地開發模式（不需 Docker）
langgraph dev

# 或用 Docker 啟動（正式模式）
langgraph up

# 測試健康檢查

# 測試執行

    "input": {
      "messages": [{"role": "human", "content": "Hello!"}]

  }'
```

## 重點摘要

| 概念 | 重點 |
|------|------|
| **Agent Server** | 標準化 API Runtime，自動處理 persistence、state、streaming |
| **CLI** | `langgraph dev` 快速啟動開發、`langgraph build` 建構 Docker image |
| **Studio** | 視覺化 IDE，支援 Graph Mode 和 Chat Mode |
| **SDK** | 統一的客戶端介面，支援同步/非同步操作 |
| **langgraph.json** | 核心設定檔，定義 dependencies、graphs、auth、store |
| **RemoteGraph** | 用 CompiledGraph 相同 API 存取遠端部署的 graph |
| **Agent Server 自動化** | 部署時不需手動設定 checkpointer 或 store |
