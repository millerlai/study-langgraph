## 1. 依節點過濾串流輸出

在複雜的圖中，你可能只想看到特定節點的輸出，而不是每個節點都接收。LangGraph 的 `stream()` 支援 `subgraphs` 參數，也可以透過檢查 chunk 中的 `langgraph_node` 欄位來過濾。

```
節點過濾示意

┌─────────┐     ┌─────────┐     ┌─────────┐
│ search  │────>│ analyze │────>│ respond │
│ (搜尋)  │     │ (分析)  │     │ (回應)  │
└─────────┘     └─────────┘     └─────────┘
     │               │               │
     │               │               ▼
     │               │          只取這個節點
     ▼               ▼          的串流輸出！
  [過濾掉]       [過濾掉]
```

### 範例：依節點名稱過濾 updates

```python
"""
依節點過濾串流輸出
只接收特定節點的 state 更新
"""
from typing import TypedDict
from langgraph.graph import StateGraph, START, END


class State(TypedDict):
    query: str
    search_result: str
    analysis: str
    response: str


def search_node(state: State) -> dict:
    return {"search_result": f"搜尋「{state['query']}」的結果：LangGraph 是圖狀態機框架"}


def analyze_node(state: State) -> dict:
    return {"analysis": f"分析結果：{state['search_result']} => 這是一個重要的 AI 工具"}


def respond_node(state: State) -> dict:
    return {"response": f"回答：根據分析，{state['analysis']}"}


graph = (
    StateGraph(State)
    .add_node("search", search_node)
    .add_node("analyze", analyze_node)
    .add_node("respond", respond_node)
    .add_edge(START, "search")
    .add_edge("search", "analyze")
    .add_edge("analyze", "respond")
    .add_edge("respond", END)
    .compile()
)

# 使用 updates 模式，只過濾 respond 節點的輸出
print("=== 只顯示 respond 節點的輸出 ===")
for chunk in graph.stream(
    {"query": "什麼是 LangGraph？"},
    stream_mode="updates",
    version="v2",
):
    # chunk["data"] 是 {node_name: state_update} 的 dict
    if "respond" in chunk["data"]:
        print(f"回應節點輸出: {chunk['data']['respond']}")

print()
print("=== 顯示所有節點（對比） ===")
for chunk in graph.stream(
    {"query": "什麼是 LangGraph？"},
    stream_mode="updates",
    version="v2",
):
    for node_name, update in chunk["data"].items():
        print(f"[{node_name}] {update}")
```

**預期輸出：**
```
=== 只顯示 respond 節點的輸出 ===
回應節點輸出: {'response': '回答：根據分析，分析結果：搜尋「什麼是 LangGraph？」的結果：LangGraph 是圖狀態機框架 => 這是一個重要的 AI 工具'}

=== 顯示所有節點（對比） ===
[search] {'search_result': '搜尋「什麼是 LangGraph？」的結果：LangGraph 是圖狀態機框架'}
[analyze] {'analysis': '分析結果：搜尋「什麼是 LangGraph？」的結果：LangGraph 是圖狀態機框架 => 這是一個重要的 AI 工具'}
[respond] {'response': '回答：根據分析，分析結果：搜尋「什麼是 LangGraph？」的結果：LangGraph 是圖狀態機框架 => 這是一個重要的 AI 工具'}
```

## 2. 依 LLM 呼叫過濾（tags）

當圖中有多個節點都會呼叫 LLM 時，你可以用 **tags** 來區分不同的 LLM 呼叫，然後在串流時依 tag 過濾。這在使用 `messages` 模式串流 LLM token 時特別有用。

```
Tags 過濾示意

┌──────────────────┐     ┌──────────────────┐
│  draft_node      │────>│  review_node     │
│  tag="drafting"  │     │  tag="reviewing" │
│  呼叫 LLM 寫初稿│     │  呼叫 LLM 審稿  │
└──────────────────┘     └──────────────────┘
         │                        │
    [messages 模式]          [messages 模式]
    metadata 含有:           metadata 含有:
    langgraph_node=          langgraph_node=
      "draft_node"             "review_node"
```

### 範例：使用 messages 模式依節點過濾 LLM token

```python
"""
依 LLM 呼叫過濾 token 串流
利用 messages 模式中的 metadata 來過濾特定節點的 LLM 輸出
"""
from typing import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.config import get_stream_writer


class State(TypedDict):
    topic: str
    draft: str
    review: str


def draft_node(state: State) -> dict:
    """模擬 LLM 寫初稿（此處用 get_stream_writer 模擬 token 串流）"""
    writer = get_stream_writer()
    # 模擬自定義串流：實際場景中，LLM 的 token 會透過 messages 模式自動串流
    writer({"status": "drafting", "node": "draft_node"})
    return {"draft": f"初稿：{state['topic']} 是一個強大的框架"}


def review_node(state: State) -> dict:
    """模擬 LLM 審稿"""
    writer = get_stream_writer()
    writer({"status": "reviewing", "node": "review_node"})
    return {"review": f"審稿通過：{state['draft']} — 內容正確"}


graph = (
    StateGraph(State)
    .add_node("draft", draft_node)
    .add_node("review", review_node)
    .add_edge(START, "draft")
    .add_edge("draft", "review")
    .add_edge("review", END)
    .compile()
)

# 同時使用 updates + custom 模式，並過濾特定節點的 custom 輸出
print("=== 只顯示 review 節點的 custom 串流 ===")
for chunk in graph.stream(
    {"topic": "LangGraph"},
    stream_mode=["updates", "custom"],
    version="v2",
):
    if chunk["type"] == "custom":
        # 依自定義 metadata 過濾
        if chunk["data"].get("node") == "review_node":
            print(f"  審稿狀態: {chunk['data']}")
    elif chunk["type"] == "updates":
        for node_name, update in chunk["data"].items():
            if node_name == "review":
                print(f"  審稿結果: {update}")
```

**預期輸出：**
```
=== 只顯示 review 節點的 custom 串流 ===
  審稿狀態: {'status': 'reviewing', 'node': 'review_node'}
  審稿結果: {'review': '審稿通過：初稿：LangGraph 是一個強大的框架 — 內容正確'}
```

> 📄 完整範例程式碼：[6.2-example-filter-by-tags.py](./6.2-example-filter-by-tags.py)

> **提示**：在真實的 LLM 串流中，使用 `stream_mode="messages"` 時，每個 chunk 的 metadata 會包含 `langgraph_node` 欄位，你可以直接用它來過濾：
> ```python
> for chunk in graph.stream(inputs, stream_mode="messages", version="v2"):
>     if chunk["data"]["metadata"].get("langgraph_node") == "target_node":
>         print(chunk["data"])  # 只印出目標節點的 LLM token
> ```

## 3. 子圖輸出（subgraph / namespace）

當圖中包含子圖（subgraph）時，你可能想分別追蹤父圖和子圖的串流輸出。LangGraph 使用 **checkpoint namespace** 來區分不同層級的輸出，並支援 `subgraphs=True` 參數來獲取子圖的串流。

```
子圖串流示意

┌─ 父圖 (namespace="") ──────────────────────┐
│                                              │
│  ┌────────┐    ┌─ 子圖 (namespace=          │
│  │ start  │───>│   "sub:uuid") ─────────┐   │
│  │        │    │                         │   │
│  └────────┘    │  ┌───────┐  ┌───────┐  │   │
│                │  │step_a │─>│step_b │  │   │
│                │  └───────┘  └───────┘  │   │
│                │                         │   │
│                └─────────────────────────┘   │
│                          │                   │
│                    ┌─────────┐               │
│                    │  end    │               │
│                    └─────────┘               │
└──────────────────────────────────────────────┘

subgraphs=True 時可以收到子圖的每一步更新！
```

### 範例：串流子圖輸出

```python
"""
子圖串流：使用 subgraphs=True 取得子圖的逐步輸出
"""
from typing import TypedDict, Annotated
from operator import add
from langgraph.graph import StateGraph, START, END


class SubState(TypedDict):
    items: Annotated[list[str], add]


class ParentState(TypedDict):
    items: Annotated[list[str], add]
    summary: str


# === 定義子圖 ===
def sub_step_a(state: SubState) -> dict:
    return {"items": ["子圖步驟A完成"]}


def sub_step_b(state: SubState) -> dict:
    return {"items": ["子圖步驟B完成"]}


subgraph = (
    StateGraph(SubState)
    .add_node("step_a", sub_step_a)
    .add_node("step_b", sub_step_b)
    .add_edge(START, "step_a")
    .add_edge("step_a", "step_b")
    .add_edge("step_b", END)
    .compile()
)


# === 定義父圖 ===
def start_node(state: ParentState) -> dict:
    return {"items": ["父圖開始"]}


def end_node(state: ParentState) -> dict:
    return {"summary": f"完成！共 {len(state['items'])} 個步驟"}


parent_graph = (
    StateGraph(ParentState)
    .add_node("start", start_node)
    .add_node("sub", subgraph)        # 子圖作為節點
    .add_node("end", end_node)
    .add_edge(START, "start")
    .add_edge("start", "sub")
    .add_edge("sub", "end")
    .add_edge("end", END)
    .compile()
)

# 啟用 subgraphs=True 來串流子圖的每一步
print("=== 含子圖串流（subgraphs=True） ===")
for namespace, chunk in parent_graph.stream(
    {"items": []},
    stream_mode="updates",
    subgraphs=True,
):
    ns_label = "父圖" if namespace == () else f"子圖({namespace})"
    for node_name, update in chunk.items():
        print(f"  [{ns_label}] {node_name}: {update}")

print()
print("=== 不含子圖串流（預設） ===")
for chunk in parent_graph.stream(
    {"items": []},
    stream_mode="updates",
):
    for node_name, update in chunk.items():
        print(f"  {node_name}: {update}")
```

**預期輸出：**
```
=== 含子圖串流（subgraphs=True） ===
  [父圖] start: {'items': ['父圖開始']}
  [子圖(('sub:...',))] step_a: {'items': ['子圖步驟A完成']}
  [子圖(('sub:...',))] step_b: {'items': ['子圖步驟B完成']}
  [父圖] sub: {'items': ['子圖步驟A完成', '子圖步驟B完成']}
  [父圖] end: {'summary': '完成！共 3 個步驟'}

=== 不含子圖串流（預設） ===
  start: {'items': ['父圖開始']}
  sub: {'items': ['子圖步驟A完成', '子圖步驟B完成']}
  end: {'summary': '完成！共 3 個步驟'}
```

> 📄 完整範例程式碼：[6.2-example-subgraph-stream.py](./6.2-example-subgraph-stream.py)

> **注意**：預設不使用 `subgraphs=True` 時，子圖的內部步驟會被合併為一個父圖節點的單一更新。啟用後才能看到子圖內部的逐步執行。

## 4. 多模式同時串流

LangGraph 的一大特色是可以**同時啟用多種串流模式**。只要傳入一個 list 給 `stream_mode`，就能在同一個迴圈中接收不同類型的串流資料。

```
多模式同時串流

┌──────────────────────────────────────────┐
│          graph.stream(                    │
│            inputs,                        │
│            stream_mode=["updates",        │
│                         "custom",         │
│                         "messages"],      │
│            version="v2"                   │
│          )                                │
│                                           │
│  chunk["type"] 區分不同模式：             │
│                                           │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ │
│  │ updates  │ │ custom   │ │ messages │ │
│  │ 狀態更新 │ │ 自定義   │ │ LLM      │ │
│  │          │ │ 事件     │ │ token    │ │
│  └──────────┘ └──────────┘ └──────────┘ │
└──────────────────────────────────────────┘
```

### 範例：同時使用 updates + custom

```python
"""
多模式同時串流
同時接收 updates（狀態變更）和 custom（自定義事件）
"""
from typing import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.config import get_stream_writer


class State(TypedDict):
    text: str
    processed: str


def preprocess(state: State) -> dict:
    writer = get_stream_writer()
    writer({"stage": "preprocessing", "progress": "50%"})
    writer({"stage": "preprocessing", "progress": "100%"})
    return {"processed": state["text"].upper()}


def postprocess(state: State) -> dict:
    writer = get_stream_writer()
    writer({"stage": "postprocessing", "progress": "100%"})
    return {"processed": f"[完成] {state['processed']}"}


graph = (
    StateGraph(State)
    .add_node("preprocess", preprocess)
    .add_node("postprocess", postprocess)
    .add_edge(START, "preprocess")
    .add_edge("preprocess", "postprocess")
    .add_edge("postprocess", END)
    .compile()
)

print("=== 多模式同時串流：updates + custom ===")
for chunk in graph.stream(
    {"text": "hello langgraph"},
    stream_mode=["updates", "custom"],
    version="v2",
):
    if chunk["type"] == "custom":
        print(f"  [進度] {chunk['data']}")
    elif chunk["type"] == "updates":
        for node_name, update in chunk["data"].items():
            print(f"  [狀態] {node_name} => {update}")
```

**預期輸出：**
```
=== 多模式同時串流：updates + custom ===
  [進度] {'stage': 'preprocessing', 'progress': '50%'}
  [進度] {'stage': 'preprocessing', 'progress': '100%'}
  [狀態] preprocess => {'processed': 'HELLO LANGGRAPH'}
  [進度] {'stage': 'postprocessing', 'progress': '100%'}
  [狀態] postprocess => {'processed': '[完成] HELLO LANGGRAPH'}
```

> 📄 完整範例程式碼：[6.2-example-multi-mode.py](./6.2-example-multi-mode.py)

## 5. 非同步串流 astream()

LangGraph 提供了 `astream()` 非同步版本，用於 `async for` 迴圈中。在 Web 伺服器（FastAPI、Starlette）或 Jupyter Notebook 等非同步環境中，`astream()` 是首選。

```
同步 vs 非同步串流

同步 (stream):
  for chunk in graph.stream(inputs):
      process(chunk)

非同步 (astream):
  async for chunk in graph.astream(inputs):
      await process(chunk)

┌────────────────────────────────────────────┐
│  適用場景                                   │
│                                             │
│  stream()  ─── CLI 工具、腳本、同步程式     │
│  astream() ─── FastAPI、Notebook、非同步程式│
└────────────────────────────────────────────┘
```

### 範例：基本 astream() 用法

```python
"""
非同步串流 astream()
使用 async for 來逐步接收串流輸出
"""
import asyncio
from typing import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.config import get_stream_writer


class State(TypedDict):
    question: str
    answer: str


def think(state: State) -> dict:
    writer = get_stream_writer()
    writer({"status": "思考中..."})
    return {"answer": f"關於「{state['question']}」的回答：LangGraph 很棒！"}


graph = (
    StateGraph(State)
    .add_node("think", think)
    .add_edge(START, "think")
    .add_edge("think", END)
    .compile()
)


async def main():
    print("=== 非同步串流 astream() ===")
    async for chunk in graph.astream(
        {"question": "什麼是 LangGraph？"},
        stream_mode=["updates", "custom"],
        version="v2",
    ):
        if chunk["type"] == "custom":
            print(f"  [狀態] {chunk['data']}")
        elif chunk["type"] == "updates":
            for node_name, update in chunk["data"].items():
                print(f"  [結果] {node_name}: {update}")


# 執行非同步函式
asyncio.run(main())
```

**預期輸出：**
```
=== 非同步串流 astream() ===
  [狀態] {'status': '思考中...'}
  [結果] think: {'answer': "關於「什麼是 LangGraph？」的回答：LangGraph 很棒！"}
```

> 📄 完整範例程式碼：[6.2-example-async-stream.py](./6.2-example-async-stream.py)

```python
"""
FastAPI + LangGraph astream() 整合
透過 Server-Sent Events 將串流推送給前端
（需安裝 fastapi, uvicorn, sse-starlette）
"""
import json
from typing import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.config import get_stream_writer

# ── 圖定義 ──
class State(TypedDict):
    query: str
    result: str


def process_query(state: State) -> dict:
    writer = get_stream_writer()
    writer({"status": "processing"})
    return {"result": f"已處理: {state['query']}"}


graph = (
    StateGraph(State)
    .add_node("process", process_query)
    .add_edge(START, "process")
    .add_edge("process", END)
    .compile()
)

# ── FastAPI 應用 ──
# from fastapi import FastAPI
# from fastapi.responses import StreamingResponse
#
# app = FastAPI()
#
# async def event_generator(query: str):
#     async for chunk in graph.astream(

#         stream_mode=["updates", "custom"],

#     ):
#         yield f"data: {json.dumps(chunk, ensure_ascii=False)}\n\n"
#     yield "data: [DONE]\n\n"
#
# @app.get("/stream")
# async def stream_endpoint(query: str = "test"):
#     return StreamingResponse(
#         event_generator(query),
#         media_type="text/event-stream",

# 獨立測試（不啟動伺服器）
import asyncio

async def test_stream():

        print(f"  data: {sse_data}")
    print("  data: [DONE]")

asyncio.run(test_stream())
```

## 6. 重點摘要

| 主題 | 說明 | 關鍵 API / 參數 |
|------|------|-----------------|
| 依節點過濾 | 在 `updates` 模式下檢查 `chunk["data"]` 的 key 來過濾特定節點 | `stream_mode="updates"` |
| 依 LLM 過濾 (tags) | 在 `messages` 模式下用 `metadata["langgraph_node"]` 過濾 | `stream_mode="messages"` |
| 子圖輸出 | 使用 `subgraphs=True` 來取得子圖的逐步串流 | `subgraphs=True` |
| 多模式同時串流 | 傳入 list 給 `stream_mode`，用 `chunk["type"]` 區分 | `stream_mode=["updates", "custom"]` |
| 非同步串流 | 使用 `astream()` + `async for`，適合 Web / Notebook | `graph.astream()` |
| SSE 整合 | 搭配 FastAPI `StreamingResponse` 推送串流 | `text/event-stream` |
| version="v2" | 推薦使用 v2 格式，chunk 為 dict 含 `type` 和 `data` | `version="v2"` |
