## 1. 總覽：四大 State 操作

當圖配備了 checkpointer，你可以在圖**執行之外**對 State 進行查詢和修改。這四個操作是 Human-in-the-Loop、Time Travel、除錯等進階功能的基礎。

```
┌─────────────────────────────────────────────────────────────────┐
│                    State 操作一覽                                │
├──────────────────┬──────────────────────────────────────────────┤
│ get_state        │ 取得指定 thread 的最新（或特定）checkpoint     │
├──────────────────┼──────────────────────────────────────────────┤
│ get_state_history│ 取得指定 thread 的所有 checkpoint 歷史        │
├──────────────────┼──────────────────────────────────────────────┤
│ update_state     │ 手動更新 State，建立一個新的 checkpoint        │
├──────────────────┼──────────────────────────────────────────────┤
│ Pending Writes   │ 節點失敗時，已成功的節點寫入會被保留，恢復時     │
│                  │ 不需重新執行成功的節點                         │
└──────────────────┴──────────────────────────────────────────────┘
```

### 操作流程圖

```
            graph.invoke(input, config)
                       │
                       v
              ┌─────────────────┐
              │  圖開始執行       │
              │  每個 super-step  │──── 自動建立 Checkpoint
              │  產生一個 CP      │
              └────────┬────────┘
                       │
                       v
        ┌──────────────┼──────────────┐
        │              │              │
        v              v              v
  get_state()    get_state_      update_state()
  取得最新 CP    history()        建立新 CP
                 取得歷史
                       │
                       v
              ┌─────────────────┐
              │ 找到特定 CP 後    │
              │ 可以用它的 config │
              │ 繼續 invoke      │
              └─────────────────┘
```

## 2. get_state：取得目前 State

### 功能

取得指定 thread 的**最新** checkpoint，或指定 `checkpoint_id` 取得特定的 checkpoint。回傳 `StateSnapshot` 物件。

### 基本用法

```python
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import InMemorySaver
from typing import TypedDict, Annotated
from operator import add

# === 1. 定義 State 和節點 ===
class State(TypedDict):
    messages: Annotated[list[str], add]
    step_count: int

def step_a(state: State) -> dict:
    return {"messages": ["step_a 完成"], "step_count": 1}

def step_b(state: State) -> dict:
    return {"messages": ["step_b 完成"], "step_count": 2}

# === 2. 建構圖 ===
builder = StateGraph(State)
builder.add_node("step_a", step_a)
builder.add_node("step_b", step_b)
builder.add_edge(START, "step_a")
builder.add_edge("step_a", "step_b")
builder.add_edge("step_b", END)

checkpointer = InMemorySaver()
graph = builder.compile(checkpointer=checkpointer)

# === 3. 執行圖 ===
config = {"configurable": {"thread_id": "get_state_demo"}}
result = graph.invoke({"messages": ["開始"], "step_count": 0}, config)

# === 4. 取得最新 State ===
snapshot = graph.get_state(config)

print("=== 最新 State ===")
print(f"values:     {snapshot.values}")
print(f"next:       {snapshot.next}")
print(f"step:       {snapshot.metadata['step']}")
print(f"source:     {snapshot.metadata['source']}")
print(f"writes:     {snapshot.metadata['writes']}")
print(f"created_at: {snapshot.created_at}")

# 預期輸出:
# === 最新 State ===
# values:     {'messages': ['開始', 'step_a 完成', 'step_b 完成'], 'step_count': 2}
# next:       ()
# step:       2
# source:     loop
# writes:     {'step_b': {'messages': ['step_b 完成'], 'step_count': 2}}
# created_at: 2024-...
```

> 📄 完整範例程式碼：[4.3-example-get-state.py](./4.3-example-get-state.py)

### 取得特定 Checkpoint

```python
# 取得最新 checkpoint 的 ID
latest_cp_id = snapshot.config["configurable"]["checkpoint_id"]
print(f"最新 checkpoint_id: {latest_cp_id}")

# 取得上一個 checkpoint（parent）
parent_cp_id = snapshot.parent_config["configurable"]["checkpoint_id"]
print(f"上一個 checkpoint_id: {parent_cp_id}")

# 用 checkpoint_id 取得特定的 checkpoint
specific_config = {
    "configurable": {
        "thread_id": "get_state_demo",
        "checkpoint_id": parent_cp_id
    }
}
parent_snapshot = graph.get_state(specific_config)
print(f"上一個 State: {parent_snapshot.values}")
print(f"上一個 next:  {parent_snapshot.next}")
# 上一個 State: {'messages': ['開始', 'step_a 完成'], 'step_count': 1}
# 上一個 next:  ('step_b',)
```

### StateSnapshot 欄位速查

```
snapshot.values         → dict       目前的 State 值
snapshot.next           → tuple      接下來要執行的節點（空 = 結束）
snapshot.config         → dict       包含 thread_id、checkpoint_id
snapshot.metadata       → dict       source、writes、step
snapshot.created_at     → str        ISO 8601 時間戳
snapshot.parent_config  → dict|None  上一個 checkpoint 的 config
snapshot.tasks          → tuple      要執行的任務清單
```

## 3. get_state_history：查詢 State 歷史

### 功能

取得指定 thread 的**所有** checkpoint 歷史。回傳一個迭代器（通常轉成 list），按**時間由新到舊**排列。

### 基本用法

```python
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import InMemorySaver
from typing import TypedDict, Annotated
from operator import add

# === 1. 定義 State 和節點 ===
class State(TypedDict):
    value: str
    history: Annotated[list[str], add]

def node_a(state: State) -> dict:
    return {"value": "a", "history": ["visited_a"]}

def node_b(state: State) -> dict:
    return {"value": "b", "history": ["visited_b"]}

def node_c(state: State) -> dict:
    return {"value": "c", "history": ["visited_c"]}

# === 2. 建構圖 ===
builder = StateGraph(State)
builder.add_node("node_a", node_a)
builder.add_node("node_b", node_b)
builder.add_node("node_c", node_c)
builder.add_edge(START, "node_a")
builder.add_edge("node_a", "node_b")
builder.add_edge("node_b", "node_c")
builder.add_edge("node_c", END)

checkpointer = InMemorySaver()
graph = builder.compile(checkpointer=checkpointer)

# === 3. 執行圖 ===
config = {"configurable": {"thread_id": "history_demo"}}
result = graph.invoke({"value": "start", "history": ["init"]}, config)

# === 4. 取得完整歷史 ===
all_states = list(graph.get_state_history(config))

print(f"共 {len(all_states)} 個 checkpoint（由新到舊）\n")

for i, state in enumerate(all_states):
    print(f"--- [{i}] step={state.metadata['step']}, "
          f"source={state.metadata['source']} ---")
    print(f"  values:  {state.values}")
    print(f"  next:    {state.next}")
    print(f"  writes:  {state.metadata['writes']}")
    print()

# 預期輸出:
# 共 5 個 checkpoint（由新到舊）
#
# --- [0] step=3, source=loop ---
#   values:  {'value': 'c', 'history': ['init', 'visited_a', 'visited_b', 'visited_c']}
#   next:    ()
#   writes:  {'node_c': {'value': 'c', 'history': ['visited_c']}}
#
# --- [1] step=2, source=loop ---
#   values:  {'value': 'b', 'history': ['init', 'visited_a', 'visited_b']}
#   next:    ('node_c',)
#   writes:  {'node_b': {'value': 'b', 'history': ['visited_b']}}
#
# --- [2] step=1, source=loop ---
#   values:  {'value': 'a', 'history': ['init', 'visited_a']}
#   next:    ('node_b',)
#   writes:  {'node_a': {'value': 'a', 'history': ['visited_a']}}
#
# --- [3] step=0, source=loop ---
#   values:  {'value': 'start', 'history': ['init']}
#   next:    ('node_a',)
#   writes:  None
#
# --- [4] step=-1, source=input ---
#   values:  {'history': []}
#   next:    ('__start__',)
#   writes:  {'value': 'start', 'history': ['init']}
```

> 📄 完整範例程式碼：[4.3-example-get-state-history.py](./4.3-example-get-state-history.py)

### 篩選特定 Checkpoint

```python
# === 找到特定條件的 checkpoint ===
history = list(graph.get_state_history(config))

# 1. 找到 node_b 執行前的 checkpoint
before_node_b = next(s for s in history if s.next == ("node_b",))
print(f"node_b 執行前: {before_node_b.values}")

# 2. 找到特定 step 的 checkpoint
step_2 = next(s for s in history if s.metadata["step"] == 2)
print(f"Step 2: {step_2.values}")

# 3. 找到所有由 update_state 建立的 checkpoint
forks = [s for s in history if s.metadata["source"] == "update"]
print(f"手動更新的 checkpoint 數量: {len(forks)}")

# 4. 從特定 checkpoint 恢復執行（Replay）
replay_result = graph.invoke(None, before_node_b.config)
print(f"從 node_b 前恢復: {replay_result}")
# node_b 和 node_c 會重新執行，node_a 不會
```

## 4. update_state：更新 State 建立新 Checkpoint

### 功能

手動修改 State 並建立一個**新的** checkpoint。這不會修改原有的 checkpoint——而是在指定的 checkpoint 上建立一個分支。

### 關鍵特性

- 更新會經過 **reducer** 處理（有 reducer 的 channel 會累加，沒有的會覆寫）
- 可透過 `as_node` 指定「假裝是哪個節點產生的」，影響下一步執行哪個節點
- 回傳新 checkpoint 的 config，可用來繼續執行

### 流程圖

```
                    原始 Checkpoint 鏈
                    ┌──────┐
                    │ CP#0 │ ← input
                    └──┬───┘
                       │
                    ┌──┴───┐
                    │ CP#1 │ ← node_a 完成
                    └──┬───┘
                       │
                    ┌──┴───┐
                    │ CP#2 │ ← node_b 完成
                    └──────┘

    呼叫 update_state(CP#1.config, values={"topic": "new"})

                    ┌──────┐
                    │ CP#0 │
                    └──┬───┘
                       │
                    ┌──┴───┐
              ┌─────│ CP#1 │─────┐
              │     └──┬───┘     │
              │        │         │
           ┌──┴───┐ ┌──┴───┐    │
           │ CP#2 │ │ CP#3 │ ← 新的分支！
           │(原始) │ │(fork)│    source="update"
           └──────┘ └──────┘
```

### 基本用法

```python
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import InMemorySaver
from typing import TypedDict, Annotated
from operator import add

# === 1. 定義 State 和節點 ===
class State(TypedDict):
    topic: str
    joke: str
    edits: Annotated[list[str], add]

def generate_topic(state: State) -> dict:
    return {"topic": "程式設計", "edits": ["topic_generated"]}

def write_joke(state: State) -> dict:
    topic = state["topic"]
    return {"joke": f"為什麼 {topic} 很有趣？因為它充滿 bug！", "edits": ["joke_written"]}

# === 2. 建構圖 ===
builder = StateGraph(State)
builder.add_node("generate_topic", generate_topic)
builder.add_node("write_joke", write_joke)
builder.add_edge(START, "generate_topic")
builder.add_edge("generate_topic", "write_joke")
builder.add_edge("write_joke", END)

checkpointer = InMemorySaver()
graph = builder.compile(checkpointer=checkpointer)

# === 3. 第一次執行 ===
config = {"configurable": {"thread_id": "update_demo"}}
result = graph.invoke({"topic": "", "joke": "", "edits": []}, config)
print(f"原始結果: {result['joke']}")
# 原始結果: 為什麼 程式設計 很有趣？因為它充滿 bug！

# === 4. 找到 write_joke 執行前的 checkpoint ===
history = list(graph.get_state_history(config))
before_joke = next(s for s in history if s.next == ("write_joke",))
print(f"write_joke 前的 topic: {before_joke.values['topic']}")
# write_joke 前的 topic: 程式設計

# === 5. 用 update_state 修改 topic，建立分支 ===
fork_config = graph.update_state(
    before_joke.config,
    values={"topic": "AI 咖啡機"},  # 修改 topic
)

# === 6. 從分支繼續執行 ===
fork_result = graph.invoke(None, fork_config)
print(f"分支結果: {fork_result['joke']}")
# 分支結果: 為什麼 AI 咖啡機 很有趣？因為它充滿 bug！

# === 7. 驗證原始歷史仍在 ===
print(f"分支的 edits: {fork_result['edits']}")
# 分支的 edits: ['topic_generated', 'joke_written']
```

### as_node 參數

`as_node` 控制 update_state 「假裝是哪個節點產生的」，從而決定下一步執行哪個節點：

```python
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import InMemorySaver
from typing import TypedDict

# === 1. 定義 State 和節點 ===
class State(TypedDict):
    value: str

def node_a(state: State) -> dict:
    return {"value": state["value"] + " -> A"}

def node_b(state: State) -> dict:
    return {"value": state["value"] + " -> B"}

def node_c(state: State) -> dict:
    return {"value": state["value"] + " -> C"}

# === 2. 建構圖: A -> B -> C ===
builder = StateGraph(State)
builder.add_node("node_a", node_a)
builder.add_node("node_b", node_b)
builder.add_node("node_c", node_c)
builder.add_edge(START, "node_a")
builder.add_edge("node_a", "node_b")
builder.add_edge("node_b", "node_c")
builder.add_edge("node_c", END)

checkpointer = InMemorySaver()
graph = builder.compile(checkpointer=checkpointer)

# === 3. 執行圖 ===
config = {"configurable": {"thread_id": "as_node_demo"}}
result = graph.invoke({"value": "Start"}, config)
print(f"完整結果: {result['value']}")
# 完整結果: Start -> A -> B -> C

# === 4. 找到 node_a 執行前的 checkpoint ===
history = list(graph.get_state_history(config))
before_a = next(s for s in history if s.next == ("node_a",))

# === 5. 使用 as_node="node_a"：假裝是 node_a 產生的，下一步執行 node_b ===
fork_config = graph.update_state(
    before_a.config,
    values={"value": "Modified"},
    as_node="node_a",  # 假裝 node_a 已經執行
)

fork_result = graph.invoke(None, fork_config)
print(f"跳過 node_a: {fork_result['value']}")
# 跳過 node_a: Modified -> B -> C
# node_a 被跳過，從 node_b 開始執行

# === 6. 使用 as_node="node_b"：假裝 node_b 也完成了 ===
fork_config2 = graph.update_state(
    before_a.config,
    values={"value": "Skip to C"},
    as_node="node_b",  # 假裝 node_a 和 node_b 都已執行
)

fork_result2 = graph.invoke(None, fork_config2)
print(f"跳到 node_c: {fork_result2['value']}")
# 跳到 node_c: Skip to C -> C
```

> 📄 完整範例程式碼：[4.3-example-as-node.py](./4.3-example-as-node.py)

### as_node 必須明確指定的三種情境

```
┌──────────────────────────────────────────────────────────────────┐
│              何時必須指定 as_node？                               │
├──────────────────┬───────────────────────────────────────────────┤
│ 平行分支         │ 多個節點在同一 step 更新 State，LangGraph 無法  │
│                  │ 判斷哪個是最後一個（會拋出 InvalidUpdateError）  │
├──────────────────┼───────────────────────────────────────────────┤
│ 全新 thread      │ 在空的 thread 上設定初始 State（常見於測試），    │
│                  │ 沒有執行歷史可以推斷                            │
├──────────────────┼───────────────────────────────────────────────┤
│ 跳過節點         │ 想讓圖「以為」某個節點已經執行完畢               │
└──────────────────┴───────────────────────────────────────────────┘
```

### update_state 與 Reducer 的互動

```python
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import InMemorySaver
from typing import TypedDict, Annotated
from operator import add

# === State 包含有 reducer 和沒有 reducer 的欄位 ===
class State(TypedDict):
    name: str                              # 無 reducer → last-write-wins
    tags: Annotated[list[str], add]        # 有 reducer → 累加

def init_node(state: State) -> dict:
    return {"name": "Alice", "tags": ["user"]}

builder = StateGraph(State)
builder.add_node("init", init_node)
builder.add_edge(START, "init")
builder.add_edge("init", END)

checkpointer = InMemorySaver()
graph = builder.compile(checkpointer=checkpointer)

config = {"configurable": {"thread_id": "reducer_demo"}}
result = graph.invoke({"name": "", "tags": []}, config)
print(f"初始: name={result['name']}, tags={result['tags']}")
# 初始: name=Alice, tags=['user']

# update_state 也會經過 reducer
fork_config = graph.update_state(
    config,
    values={"name": "Bob", "tags": ["admin"]},  # name 覆寫，tags 累加
)
updated = graph.get_state(fork_config)
print(f"更新後: name={updated.values['name']}, tags={updated.values['tags']}")
# 更新後: name=Bob, tags=['user', 'admin']
#         ^^^^       ^^^^^^^^^^^^^^^^
#         覆寫       累加（reducer=add）
```

## 5. Pending Writes：失敗時的寫入保留

### 概念

當同一個 super-step 中有**多個節點**平行執行，如果其中一個節點失敗了，**已成功完成的節點的寫入會被保留**為 pending writes。當你從該 super-step 恢復執行時，成功的節點不需重新執行。

### 流程圖

```
                     Super-step N
              ┌──────────┬──────────┐
              │          │          │
              v          v          v
           node_a     node_b     node_c
           成功 ✓      失敗 ✗      成功 ✓
              │          │          │
              v          v          v
         寫入保留     拋出錯誤    寫入保留
         (pending)               (pending)

                         │
                         v
                    圖執行暫停
                    Checkpoint 保留了：
                    - node_a 的結果 (pending write)
                    - node_c 的結果 (pending write)
                    - node_b 的錯誤

                         │
                    修復問題後恢復
                         │
                         v
                     Super-step N (恢復)
              ┌──────────┬──────────┐
              │          │          │
              v          v          v
           node_a     node_b     node_c
           跳過 ⏭     重新執行    跳過 ⏭
           (已有結果)   (修復後)   (已有結果)
```

### 完整範例：模擬失敗與恢復

```python
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import InMemorySaver
from typing import TypedDict, Annotated
from operator import add

# === 1. 定義 State ===
class State(TypedDict):
    results: Annotated[list[str], add]

# === 2. 用全域變數模擬失敗 ===
call_count = 0

def reliable_node(state: State) -> dict:
    """總是成功的節點"""
    return {"results": ["reliable 完成"]}

def flaky_node(state: State) -> dict:
    """第一次呼叫時失敗，第二次成功"""
    global call_count
    call_count += 1
    if call_count == 1:
        raise RuntimeError("暫時性錯誤！")
    return {"results": ["flaky 完成"]}

# === 3. 建構圖——兩個節點平行執行 ===
builder = StateGraph(State)
builder.add_node("reliable", reliable_node)
builder.add_node("flaky", flaky_node)

# 兩個節點都從 START 出發（平行）
builder.add_edge(START, "reliable")
builder.add_edge(START, "flaky")
builder.add_edge("reliable", END)
builder.add_edge("flaky", END)

checkpointer = InMemorySaver()
graph = builder.compile(checkpointer=checkpointer)

# === 4. 第一次執行——flaky_node 會失敗 ===
config = {"configurable": {"thread_id": "pending_demo"}}
try:
    result = graph.invoke({"results": ["開始"]}, config)
except Exception as e:
    print(f"執行失敗: {e}")
    # 執行失敗: 暫時性錯誤！

# === 5. 查看 State——reliable_node 的結果已被保留 ===
state = graph.get_state(config)
print(f"目前 values: {state.values}")
print(f"下一步: {state.next}")
# 目前 values: {'results': ['開始']}
# 下一步: ('reliable', 'flaky')

# === 6. 恢復執行——reliable_node 不會重新執行 ===
# call_count 已經是 1，第二次呼叫 flaky_node 會成功
result = graph.invoke(None, config)
print(f"恢復後結果: {result['results']}")
# 恢復後結果: ['開始', 'reliable 完成', 'flaky 完成']
# reliable_node 的 pending write 被使用，不需重新執行
```

> 📄 完整範例程式碼：[4.3-example-pending-writes.py](./4.3-example-pending-writes.py)

### Pending Writes 的關鍵行為

```
┌──────────────────────────────────────────────────────────────────┐
│                Pending Writes 關鍵行為                           │
├──────────────────┬───────────────────────────────────────────────┤
│ 儲存時機         │ 節點成功完成時立即儲存（即使同 step 其他節點失   │
│                  │ 敗）                                          │
├──────────────────┼───────────────────────────────────────────────┤
│ 恢復時           │ 已有 pending write 的節點會被跳過，直接使用之前  │
│                  │ 的結果                                        │
├──────────────────┼───────────────────────────────────────────────┤
│ 觸發條件         │ 同一個 super-step 中有多個節點平行執行時         │
├──────────────────┼───────────────────────────────────────────────┤
│ 循序圖           │ 循序圖中每個 super-step 只有一個節點，所以       │
│                  │ pending writes 主要對平行圖有意義               │
├──────────────────┼───────────────────────────────────────────────┤
│ Checkpointer     │ 需要 checkpointer 才能使用此功能                │
│ 必要性           │ 透過 .put_writes 方法儲存                      │
└──────────────────┴───────────────────────────────────────────────┘
```

## 6. 綜合範例：完整的 State 操作工作流

以下範例展示一個完整的工作流：執行圖、查看歷史、從特定 checkpoint 分支、恢復執行。

```python
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import InMemorySaver
from typing import TypedDict, Annotated
from operator import add

# =============================================
# 綜合範例：文章撰寫工作流
# 流程：選題 → 撰寫初稿 → 審校
# =============================================

# === 1. 定義 State ===
class ArticleState(TypedDict):
    topic: str
    draft: str
    review: str
    log: Annotated[list[str], add]

# === 2. 定義節點 ===
def choose_topic(state: ArticleState) -> dict:
    return {
        "topic": "LangGraph 入門",
        "log": ["[choose_topic] 選定主題: LangGraph 入門"]
    }

def write_draft(state: ArticleState) -> dict:
    topic = state["topic"]
    draft = f"《{topic}》\n\nLangGraph 是一個強大的框架..."
    return {
        "draft": draft,
        "log": [f"[write_draft] 撰寫關於 '{topic}' 的初稿"]
    }

def review_draft(state: ArticleState) -> dict:
    return {
        "review": "審校通過，內容品質良好。",
        "log": ["[review_draft] 審校完成"]
    }

# === 3. 建構圖 ===
builder = StateGraph(ArticleState)
builder.add_node("choose_topic", choose_topic)
builder.add_node("write_draft", write_draft)
builder.add_node("review_draft", review_draft)
builder.add_edge(START, "choose_topic")
builder.add_edge("choose_topic", "write_draft")
builder.add_edge("write_draft", "review_draft")
builder.add_edge("review_draft", END)

checkpointer = InMemorySaver()
graph = builder.compile(checkpointer=checkpointer)

# === 4. 執行工作流 ===
config = {"configurable": {"thread_id": "article_001"}}
result = graph.invoke(
    {"topic": "", "draft": "", "review": "", "log": []},
    config
)

print("=== 執行結果 ===")
print(f"主題: {result['topic']}")
print(f"初稿: {result['draft'][:30]}...")
print(f"審校: {result['review']}")
print(f"日誌: {result['log']}")
print()

# === 5. 查看完整歷史 ===
print("=== 完整歷史 ===")
history = list(graph.get_state_history(config))
for i, state in enumerate(history):
    step = state.metadata["step"]
    source = state.metadata["source"]
    next_nodes = state.next
    print(f"[{i}] step={step}, source={source}, next={next_nodes}")

print()

# === 6. 找到撰寫初稿前的 checkpoint ===
before_draft = next(s for s in history if s.next == ("write_draft",))
print(f"=== 撰寫初稿前的 State ===")
print(f"topic: {before_draft.values['topic']}")
print(f"draft: '{before_draft.values['draft']}'  (還是空的)")
print()

# === 7. 修改主題後分支 ===
fork_config = graph.update_state(
    before_draft.config,
    values={"topic": "LangGraph 進階技巧"},
    as_node="choose_topic",  # 假裝是 choose_topic 產生的
)

# === 8. 從分支繼續執行 ===
fork_result = graph.invoke(None, fork_config)
print("=== 分支結果 ===")
print(f"主題: {fork_result['topic']}")
print(f"初稿: {fork_result['draft'][:40]}...")
print()

# === 9. 驗證原始結果和分支結果並存 ===
print("=== 驗證：查看更新後的歷史 ===")
updated_history = list(graph.get_state_history(config))
print(f"原始歷史有 {len(history)} 個 checkpoint")
print(f"更新後有   {len(updated_history)} 個 checkpoint")

# 找到所有 source="update" 的 checkpoint
updates = [s for s in updated_history if s.metadata["source"] == "update"]
print(f"其中手動更新的: {len(updates)} 個")

# === 10. 取得目前最新 State（是分支的結果） ===
latest = graph.get_state(config)
print(f"\n最新 State 的主題: {latest.values['topic']}")
print(f"最新 State 的審校: {latest.values['review']}")
```

## 重點摘要

| 操作 | 一句話說明 | 關鍵 API |
|------|-----------|---------|
| **get_state** | 取得最新或特定 checkpoint 的 State 快照 | `graph.get_state(config)` |
| **get_state_history** | 取得 thread 的所有 checkpoint 歷史（新到舊） | `list(graph.get_state_history(config))` |
| **update_state** | 手動更新 State，建立新 checkpoint（分支） | `graph.update_state(config, values, as_node=)` |
| **as_node** | 指定 update_state 假裝是哪個節點產生的 | 影響 `next` 欄位（下一步執行哪個節點） |
| **Pending Writes** | 平行節點部分失敗時保留成功結果 | 自動運作，恢復時跳過已成功的節點 |
| **Replay** | 從某個 checkpoint 重新執行後續節點 | `graph.invoke(None, checkpoint_config)` |
| **Fork** | 從某個 checkpoint 修改後分支執行 | `update_state()` + `invoke(None, fork_config)` |

### 操作選擇指南

```
想查看目前圖的狀態？              → get_state(config)
想看圖的完整執行歷史？            → get_state_history(config)
想修改 State 然後繼續？           → update_state(config, values) + invoke(None, new_config)
想從某個歷史點重新執行？          → invoke(None, old_checkpoint.config)
想跳過某些節點？                 → update_state(..., as_node="later_node")
節點失敗想要恢復？               → 修復問題後 invoke(None, config)，pending writes 自動生效
```
