## 1. 總覽：Checkpointer 生態系

LangGraph 的持久化由 Checkpointer 物件實現。所有 Checkpointer 都遵循 `BaseCheckpointSaver` 介面，並以獨立套件提供：

```
┌─────────────────────────────────────────────────────────────────┐
│                  LangGraph Checkpointer 生態系                   │
├──────────────────────┬──────────────────────────────────────────┤
│ langgraph-checkpoint │ 基礎介面 + InMemorySaver（內含於          │
│                      │ langgraph 主套件）                        │
├──────────────────────┼──────────────────────────────────────────┤
│ langgraph-checkpoint │ SQLite 實作（SqliteSaver /               │
│ -sqlite              │ AsyncSqliteSaver），需另外安裝             │
├──────────────────────┼──────────────────────────────────────────┤
│ langgraph-checkpoint │ PostgreSQL 實作（PostgresSaver /          │
│ -postgres            │ AsyncPostgresSaver），需另外安裝           │
└──────────────────────┴──────────────────────────────────────────┘
```

## 2. BaseCheckpointSaver 介面

所有 Checkpointer 都實作以下方法：

```
┌─────────────────────────────────────────────────────────────────┐
│                BaseCheckpointSaver 介面方法                      │
├──────────────┬──────────────────────────────────────────────────┤
│ .put         │ 儲存一個 checkpoint（含 config 和 metadata）      │
├──────────────┼──────────────────────────────────────────────────┤
│ .put_writes  │ 儲存與 checkpoint 關聯的中間寫入（pending writes）  │
├──────────────┼──────────────────────────────────────────────────┤
│ .get_tuple   │ 根據 config（thread_id + checkpoint_id）取出      │
│              │ checkpoint，供 get_state() 使用                   │
├──────────────┼──────────────────────────────────────────────────┤
│ .list        │ 列出符合條件的 checkpoint，供 get_state_history() │
│              │ 使用                                             │
├──────────────┼──────────────────────────────────────────────────┤
│ 非同步版本    │ .aput、.aput_writes、.aget_tuple、.alist         │
│              │ 用於 ainvoke / astream 等非同步執行                │
└──────────────┴──────────────────────────────────────────────────┘
```

## 3. InMemorySaver（開發用）

### 特性

- **儲存位置**：Python 程式的記憶體中（dict）
- **持久性**：程式結束即消失
- **安裝**：不需額外安裝，內含於 `langgraph` 套件
- **適用場景**：開發、測試、原型驗證
- **同步/非同步**：同時支援

### 基本用法

```python
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import InMemorySaver
from typing import TypedDict, Annotated
from operator import add

# === 1. 定義 State ===
class ChatState(TypedDict):
    messages: Annotated[list[str], add]
    summary: str

# === 2. 定義節點 ===
def chat_node(state: ChatState) -> dict:
    last_msg = state["messages"][-1]
    reply = f"你說了: {last_msg}"
    return {"messages": [reply]}

def summarize_node(state: ChatState) -> dict:
    return {"summary": f"共 {len(state['messages'])} 則訊息"}

# === 3. 建構圖 ===
builder = StateGraph(ChatState)
builder.add_node("chat", chat_node)
builder.add_node("summarize", summarize_node)
builder.add_edge(START, "chat")
builder.add_edge("chat", "summarize")
builder.add_edge("summarize", END)

# === 4. 使用 InMemorySaver 編譯 ===
checkpointer = InMemorySaver()
graph = builder.compile(checkpointer=checkpointer)

# === 5. 執行多輪對話 ===
config = {"configurable": {"thread_id": "demo_001"}}

# 第一輪
result1 = graph.invoke({"messages": ["你好"], "summary": ""}, config)
print(f"第一輪: {result1['messages']}")
print(f"摘要:   {result1['summary']}")
# 第一輪: ['你好', '你說了: 你好']
# 摘要:   共 2 則訊息

# 第二輪（接續上次的 State）
result2 = graph.invoke({"messages": ["今天天氣如何？"]}, config)
print(f"第二輪: {result2['messages']}")
print(f"摘要:   {result2['summary']}")
# 第二輪: ['你好', '你說了: 你好', '今天天氣如何？', '你說了: 今天天氣如何？']
# 摘要:   共 4 則訊息
```

### 注意事項

```
⚠️  InMemorySaver 僅適用於開發與測試：
    - 程式重啟後所有資料消失
    - 不支援多 process / 多 server 共享
    - 記憶體用量會隨 checkpoint 數量線性增長
```

## 4. SqliteSaver（本地持久化）

### 特性

- **儲存位置**：SQLite 資料庫檔案（或 `:memory:`）
- **持久性**：檔案存在即保留
- **安裝**：需額外安裝 `langgraph-checkpoint-sqlite`
- **適用場景**：本地開發、單機應用、原型展示
- **同步/非同步**：`SqliteSaver`（同步）/ `AsyncSqliteSaver`（非同步）

### 安裝

```bash
uv add langgraph-checkpoint-sqlite
```

### 同步用法

```python
import sqlite3
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.sqlite import SqliteSaver
from typing import TypedDict, Annotated
from operator import add

# === 1. 定義 State 和節點 ===
class State(TypedDict):
    messages: Annotated[list[str], add]

def echo(state: State) -> dict:
    return {"messages": [f"Echo: {state['messages'][-1]}"]}

# === 2. 建構圖 ===
builder = StateGraph(State)
builder.add_node("echo", echo)
builder.add_edge(START, "echo")
builder.add_edge("echo", END)

# === 3. 使用 SqliteSaver（以 context manager 管理連線） ===
with SqliteSaver.from_conn_string("checkpoints.db") as checkpointer:
    graph = builder.compile(checkpointer=checkpointer)

    config = {"configurable": {"thread_id": "sqlite_demo"}}

    # 第一次執行
    r1 = graph.invoke({"messages": ["第一次"]}, config)
    print(r1["messages"])
    # ['第一次', 'Echo: 第一次']

    # 第二次執行（接續）
    r2 = graph.invoke({"messages": ["第二次"]}, config)
    print(r2["messages"])
    # ['第一次', 'Echo: 第一次', '第二次', 'Echo: 第二次']

# === 4. 也可以直接傳入 sqlite3 連線 ===
conn = sqlite3.connect("checkpoints.db")
checkpointer = SqliteSaver(conn)
graph = builder.compile(checkpointer=checkpointer)

config = {"configurable": {"thread_id": "sqlite_demo"}}
state = graph.get_state(config)
print(f"之前的訊息仍在: {state.values['messages']}")
# 之前的訊息仍在: ['第一次', 'Echo: 第一次', '第二次', 'Echo: 第二次']

conn.close()
```

### 非同步用法

```python
import asyncio
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.sqlite.aio import AsyncSqliteSaver
from typing import TypedDict, Annotated
from operator import add

# === 1. 定義 State 和節點 ===
class State(TypedDict):
    messages: Annotated[list[str], add]

def echo(state: State) -> dict:
    return {"messages": [f"Echo: {state['messages'][-1]}"]}

# === 2. 建構圖 ===
builder = StateGraph(State)
builder.add_node("echo", echo)
builder.add_edge(START, "echo")
builder.add_edge("echo", END)

# === 3. 非同步執行 ===
async def main():
    async with AsyncSqliteSaver.from_conn_string("async_checkpoints.db") as checkpointer:
        graph = builder.compile(checkpointer=checkpointer)

        config = {"configurable": {"thread_id": "async_demo"}}

        r1 = await graph.ainvoke({"messages": ["非同步第一次"]}, config)
        print(r1["messages"])

        r2 = await graph.ainvoke({"messages": ["非同步第二次"]}, config)
        print(r2["messages"])

asyncio.run(main())
```

## 5. PostgresSaver（正式環境）

### 特性

- **儲存位置**：PostgreSQL 資料庫
- **持久性**：永久保存
- **安裝**：需額外安裝 `langgraph-checkpoint-postgres`
- **適用場景**：正式環境、多伺服器共享、高可用場景
- **同步/非同步**：`PostgresSaver`（同步）/ `AsyncPostgresSaver`（非同步）

### 安裝

```bash
uv add langgraph-checkpoint-postgres
```

### 同步用法

```python
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.postgres import PostgresSaver
from typing import TypedDict, Annotated
from operator import add

# === 1. 定義 State 和節點 ===
class State(TypedDict):
    messages: Annotated[list[str], add]

def echo(state: State) -> dict:
    return {"messages": [f"Echo: {state['messages'][-1]}"]}

# === 2. 建構圖 ===
builder = StateGraph(State)
builder.add_node("echo", echo)
builder.add_edge(START, "echo")
builder.add_edge("echo", END)

# === 3. 使用 PostgresSaver ===
DB_URI = "postgresql://user:password@localhost:5432/langgraph_db"

with PostgresSaver.from_conn_string(DB_URI) as checkpointer:
    # 首次使用需要建立資料表
    checkpointer.setup()

    graph = builder.compile(checkpointer=checkpointer)

    config = {"configurable": {"thread_id": "postgres_demo"}}

    r1 = graph.invoke({"messages": ["Hello Postgres"]}, config)
    print(r1["messages"])
    # ['Hello Postgres', 'Echo: Hello Postgres']
```

### 非同步用法

```python
import asyncio
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.postgres.aio import AsyncPostgresSaver
from typing import TypedDict, Annotated
from operator import add

# === 1. 定義 State 和節點 ===
class State(TypedDict):
    messages: Annotated[list[str], add]

def echo(state: State) -> dict:
    return {"messages": [f"Echo: {state['messages'][-1]}"]}

# === 2. 建構圖 ===
builder = StateGraph(State)
builder.add_node("echo", echo)
builder.add_edge(START, "echo")
builder.add_edge("echo", END)

# === 3. 非同步 PostgresSaver ===
DB_URI = "postgresql://user:password@localhost:5432/langgraph_db"

async def main():
    async with AsyncPostgresSaver.from_conn_string(DB_URI) as checkpointer:
        await checkpointer.asetup()

        graph = builder.compile(checkpointer=checkpointer)

        config = {"configurable": {"thread_id": "async_pg_demo"}}

        r1 = await graph.ainvoke({"messages": ["Async Postgres"]}, config)
        print(r1["messages"])

asyncio.run(main())
```

### PostgresSaver 生產環境注意事項

```
┌─────────────────────────────────────────────────────────────────┐
│                PostgresSaver 正式環境檢查清單                     │
├──────────────┬──────────────────────────────────────────────────┤
│ setup()      │ 首次使用需呼叫 checkpointer.setup() 建立表結構    │
├──────────────┼──────────────────────────────────────────────────┤
│ 連線池       │ 正式環境建議使用 connection pool 管理連線          │
├──────────────┼──────────────────────────────────────────────────┤
│ 非同步       │ Web 應用建議使用 AsyncPostgresSaver              │
├──────────────┼──────────────────────────────────────────────────┤
│ 備份         │ 定期備份資料庫以防資料遺失                        │
├──────────────┼──────────────────────────────────────────────────┤
│ 清理         │ 考慮定期清理過期的 thread / checkpoint            │
└──────────────┴──────────────────────────────────────────────────┘
```

## 6. Checkpoint Namespace（父圖/子圖）

### 定義

每個 checkpoint 有一個 `checkpoint_ns`（checkpoint namespace）欄位，用來識別該 checkpoint 屬於哪個圖或子圖。

### Namespace 規則

| 情境 | checkpoint_ns 值 | 說明 |
|------|------------------|------|
| 父圖（根圖） | `""` （空字串） | 最外層的圖 |
| 子圖 | `"node_name:uuid"` | 以子圖所在的節點名稱 + UUID 組成 |
| 巢狀子圖 | `"outer:uuid\|inner:uuid"` | 多層子圖用 `\|` 分隔 |

### 在節點中存取 Namespace

```python
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import InMemorySaver
from langchain_core.runnables import RunnableConfig
from typing import TypedDict

# === 1. 定義 State ===
class State(TypedDict):
    value: str

# === 2. 定義節點——透過 config 取得 namespace ===
def my_node(state: State, config: RunnableConfig) -> dict:
    checkpoint_ns = config["configurable"].get("checkpoint_ns", "")
    if checkpoint_ns == "":
        print("  我在父圖中執行")
    else:
        print(f"  我在子圖中執行，namespace: {checkpoint_ns}")
    return {"value": f"processed in ns={checkpoint_ns}"}

# === 3. 建立子圖 ===
sub_builder = StateGraph(State)
sub_builder.add_node("sub_node", my_node)
sub_builder.add_edge(START, "sub_node")
sub_builder.add_edge("sub_node", END)
subgraph = sub_builder.compile()

# === 4. 建立父圖（包含子圖作為節點） ===
parent_builder = StateGraph(State)
parent_builder.add_node("parent_node", my_node)
parent_builder.add_node("child_graph", subgraph)
parent_builder.add_edge(START, "parent_node")
parent_builder.add_edge("parent_node", "child_graph")
parent_builder.add_edge("child_graph", END)

checkpointer = InMemorySaver()
parent_graph = parent_builder.compile(checkpointer=checkpointer)

# === 5. 執行 ===
config = {"configurable": {"thread_id": "ns_demo"}}
result = parent_graph.invoke({"value": "hello"}, config)

# 輸出:
#   我在父圖中執行
#   我在子圖中執行，namespace: child_graph:xxxxxxxx-xxxx-...
```

### 子圖 Checkpointer 模式

```
┌──────────────────────────────────────────────────────────────────┐
│                    子圖 Checkpointer 模式                        │
├────────────────┬─────────────────────────────────────────────────┤
│ 繼承父圖       │ subgraph.compile()                             │
│ (預設)         │ 子圖整體視為父圖的一個 super-step               │
│                │ 無法單獨 time travel 到子圖內部                  │
├────────────────┼─────────────────────────────────────────────────┤
│ 獨立 Checkpoint│ subgraph.compile(checkpointer=True)            │
│                │ 子圖每一步都有自己的 checkpoint                  │
│                │ 可以 time travel 到子圖內部的任何步驟             │
├────────────────┼─────────────────────────────────────────────────┤
│ 無 Checkpoint  │ subgraph.compile(checkpointer=False)           │
│                │ 完全不做 checkpointing，如同一般函數              │
└────────────────┴─────────────────────────────────────────────────┘
```

### 子圖獨立 Checkpoint 範例

```python
from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import InMemorySaver
from typing import TypedDict, Annotated
from operator import add

# === 1. 定義 State ===
class State(TypedDict):
    steps: Annotated[list[str], add]

# === 2. 定義子圖節點 ===
def sub_step_a(state: State) -> dict:
    return {"steps": ["sub_a 完成"]}

def sub_step_b(state: State) -> dict:
    return {"steps": ["sub_b 完成"]}

# === 3. 建立有獨立 checkpointer 的子圖 ===
sub_builder = StateGraph(State)
sub_builder.add_node("sub_step_a", sub_step_a)
sub_builder.add_node("sub_step_b", sub_step_b)
sub_builder.add_edge(START, "sub_step_a")
sub_builder.add_edge("sub_step_a", "sub_step_b")
sub_builder.add_edge("sub_step_b", END)

# checkpointer=True 表示子圖使用獨立 checkpoint
subgraph = sub_builder.compile(checkpointer=True)

# === 4. 建立父圖 ===
def parent_step(state: State) -> dict:
    return {"steps": ["parent 完成"]}

parent_builder = StateGraph(State)
parent_builder.add_node("parent_step", parent_step)
parent_builder.add_node("my_subgraph", subgraph)
parent_builder.add_edge(START, "parent_step")
parent_builder.add_edge("parent_step", "my_subgraph")
parent_builder.add_edge("my_subgraph", END)

checkpointer = InMemorySaver()
parent_graph = parent_builder.compile(checkpointer=checkpointer)

# === 5. 執行並查看子圖 checkpoint ===
config = {"configurable": {"thread_id": "subgraph_cp_demo"}}
result = parent_graph.invoke({"steps": ["開始"]}, config)
print(result["steps"])
# ['開始', 'parent 完成', 'sub_a 完成', 'sub_b 完成']

# 取得父圖 state（包含子圖資訊）
parent_state = parent_graph.get_state(config, subgraphs=True)
print(f"父圖 checkpoint_ns: '{parent_state.config['configurable']['checkpoint_ns']}'")
# 父圖 checkpoint_ns: ''
```

## 7. 序列化與加密

### 預設序列化器

LangGraph 使用 `JsonPlusSerializer` 作為預設序列化器，底層使用 `ormsgpack` 和 JSON。它支援大部分常見型別，包括 LangChain/LangGraph 原語、datetime、enum 等。

### Pickle Fallback

如果你的 State 包含 `JsonPlusSerializer` 不支援的型別（如 Pandas DataFrame），可以啟用 pickle fallback：

```python
from langgraph.checkpoint.memory import InMemorySaver
from langgraph.checkpoint.serde.jsonplus import JsonPlusSerializer

# 啟用 pickle fallback
checkpointer = InMemorySaver(
    serde=JsonPlusSerializer(pickle_fallback=True)
)
graph = builder.compile(checkpointer=checkpointer)
```

### 加密

Checkpointer 可選擇加密所有持久化的 State：

```python
import sqlite3
from langgraph.checkpoint.serde.encrypted import EncryptedSerializer
from langgraph.checkpoint.sqlite import SqliteSaver

# 從環境變數 LANGGRAPH_AES_KEY 讀取 AES 金鑰
serde = EncryptedSerializer.from_pycryptodome_aes()
checkpointer = SqliteSaver(sqlite3.connect("encrypted.db"), serde=serde)
```

## 8. Checkpointer 選擇比較

| 特性 | InMemorySaver | SqliteSaver | PostgresSaver |
|------|:------------:|:-----------:|:------------:|
| **安裝** | 內含 | `langgraph-checkpoint-sqlite` | `langgraph-checkpoint-postgres` |
| **持久性** | 程式結束即消失 | 檔案存在即保留 | 永久保存 |
| **多 process 共享** | 不支援 | 有限支援 | 完整支援 |
| **非同步支援** | 內建 | AsyncSqliteSaver | AsyncPostgresSaver |
| **需要 setup()** | 否 | 否 | 是 |
| **適用場景** | 開發/測試 | 本地/原型 | 正式環境 |
| **效能** | 最快（記憶體） | 中等 | 取決於資料庫配置 |
| **可擴展性** | 單 process | 單機 | 多伺服器 |

### 選擇指南

```
只是在本地開發/測試？                → InMemorySaver
需要本地持久化但不需要多伺服器？      → SqliteSaver
正式環境、多伺服器、高可用？          → PostgresSaver
使用 LangGraph Agent Server？       → 自動處理，不需手動設定
需要加密 State？                    → 任何 Checkpointer + EncryptedSerializer
```

## 重點摘要

| 概念 | 一句話說明 | 關鍵 API |
|------|-----------|---------|
| **BaseCheckpointSaver** | 所有 Checkpointer 的基礎介面 | `.put`, `.get_tuple`, `.list`, `.put_writes` |
| **InMemorySaver** | 記憶體內 checkpointer，開發用 | `from langgraph.checkpoint.memory import InMemorySaver` |
| **SqliteSaver** | SQLite 檔案 checkpointer，本地用 | `SqliteSaver.from_conn_string("file.db")` |
| **PostgresSaver** | PostgreSQL checkpointer，正式環境 | `PostgresSaver.from_conn_string(uri)` + `.setup()` |
| **checkpoint_ns** | 識別 checkpoint 屬於父圖或子圖 | `""` = 父圖，`"node:uuid"` = 子圖 |
| **checkpointer=True** | 子圖使用獨立 checkpoint 歷史 | `subgraph.compile(checkpointer=True)` |
| **EncryptedSerializer** | 加密所有持久化 State | `EncryptedSerializer.from_pycryptodome_aes()` |
